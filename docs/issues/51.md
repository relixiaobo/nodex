# #51 新建节点首次输入后需按两次 Enter 才创建下一节点

**状态**: 📋 待认领  
**负责人**: —  
**优先级**: P1  
**关联**: editor / trailing input / enter behavior

---

## 问题描述

对于“新建出来并首次输入文本”的节点，按一次 `Enter` 不会立即创建下一节点，需要按第二次 `Enter` 才生效。  
同样操作在“已有文本的普通节点”上是一次 `Enter` 即可，行为不一致。

## 复现步骤

1. 创建一个全新节点（初始为空）。
2. 输入任意文本（如 `test`）。
3. 按一次 `Enter`。
4. 观察是否创建下一节点。

## 预期结果

- 一次 `Enter` 立即创建下一个节点（与普通节点一致）。

## 实际结果

- 第一次 `Enter` 无创建；
- 第二次 `Enter` 才创建。

## 影响

- 破坏核心输入流，用户明显感知为“键盘响应不稳定”。

## 相关代码

- `src/components/editor/NodeEditor.tsx`
  - `handleKeyDown`（Enter 分支）
  - `saveContent` / `savedRef` / `triggerStateRef`
  - composition 相关逻辑
- `src/components/editor/TrailingInput.tsx`
  - 新建节点与 focus 交接
- `src/components/outliner/OutlinerItem.tsx`

## 建议接手方向

1. 把“新建节点首次输入”与“普通节点输入”统一到同一 Enter 决策路径。
2. 检查首次输入后是否被错误标记为 composing / dropdown-active / saved。
3. 给该场景补独立回归测试，确保：
   - 新建节点首次输入后单次 Enter 生效；
   - 已有节点行为不回退。
