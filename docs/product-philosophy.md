# soma 产品方法论 v2

> Think where you read.

---

## 一、我们相信什么

### 笔记的唯一价值是"有用"

如果记下来的东西在未来用不上，没有人会坚持记录。工具的职责不是让用户记更多笔记，而是让每一条笔记都有更高的概率在未来被用上。

笔记有用的前提有两个：
- **笔记本身被思考过** — 用自己的话写、经过组织和连接，而非原文复制粘贴
- **在需要的时候能发挥价值** — 不依赖用户记住"我记过这个"，而是在正确的时刻主动浮现，甚至直接帮你解决问题

### 不要 think later，要 think now

记笔记不是存档，而是思考。你在网上读了很多，但大部分都忘了——因为你当时没有停下来思考，或者思考了但没有记录下来，并且与自己的其他知识产生连接。

把信息用自己的话重新表述的过程，本身就是理解和记忆的过程。不经过思考的信息，未来大概率用不上——要么想不起来，要么找到了也需要重新理解一遍。

### 工具应该让人变强，而不是让人觉得工具很强

解构外部信息（总结、翻译、提取），用户会觉得"工具很强"。帮助用户建构自己的想法（组织、连接、应用），用户才会觉得"我变强了"。soma 追求后者。

## 二、方法论：Think → Connect → Compound

soma 的知识工作流是一个三步循环，每一步都让下一步更有价值：

```
Think（读到就想，想到就写）
  ↓
Connect（在笔记之间发现脉络）
  ↓
Compound（积累的思考产生复利）
  ↓
  └──→ 感受到价值 ──→ 更愿意 Think ...
```

### Think — 读到就想，想到就写

> "务必立即记下乍现的灵光与感动，一旦错失，便难以重觅。"

阅读的那一刻，就是思考的最佳时刻。soma 活在浏览器侧边栏里——你在哪里阅读，就在哪里思考。不需要切换应用，不需要"稍后处理"，你的想法就在它的来源旁边。

Think 的核心不是"记录"，而是**用自己的话写**。这是费曼学习法的实践——如果你不能用自己的话表述一个想法，说明你还没有真正理解它。写下来的过程本身就是理解。

#### 高亮与笔记的关系

高亮是 Think 的第一步——选中一段话意味着"这对我有意义"。这个判断本身就是思考，不需要立刻用文字表达。

```
#source（网页）
 ├── #highlight                  ← 纯高亮，捕捉阶段（等待后续整理）
 ├── #note                       ← 用户的想法是主角
 │    └── #highlight             ← 触发思考的原文是证据
 └── #note
      └── #highlight
```

**笔记和高亮是同级节点**，都是 `#source` 的子节点。有 note 时，`#note` 是外层（用户的思考是主角），`#highlight` 作为其子节点（原文是证据）。没有 note 时，`#highlight` 直接挂在 `#source` 下，等待后续整理。

**捕捉零门槛，反思可延迟。** 高亮即保存，note 被鼓励但不强制。用户在后续整理高亮时补写 note——这个整理过程本身就是 Connect 阶段，是建立与已有知识连接、产生新理解的时刻。

#### Think 的行为准则

- **用自己的话写** — 核心是先理解，再记录，而不是鹦鹉学舌
- **删除是日常动作** — 认同和触动不等于有用，只保留有用的
- **日记即默认** — 所有 Think 先进入 Today，不强迫分类
- **剪藏即结构** — 网页内容自动带上来源、日期、标签

### Connect — 在笔记之间发现脉络

> "组织笔记的过程带来了理解的必要难度，也许思考才是最重要的记笔记的原因。"

Think 让单条笔记有价值，Connect 让笔记之间产生关系。这不是"整理"，是**发现**——发现你的想法之间隐藏的脉络和结构。

soma 用大纲作为 Connect 的介质。大纲的每一个操作都是一种认知行为：

| 操作 | 你在做的事 |
|------|-----------|
| 把 A 拖到 B 下面 | 发现 A 属于 B — 分类 |
| 调整兄弟节点顺序 | 判断优先级 — 这个比那个重要 |
| 给节点打标签 | 定义 — 这是一个 X |
| 创建引用链接 | 建立连接 — 这和那个有关 |
| 折叠一组子节点 | 聚焦 — 这部分我理解了 |
| 删除一条笔记 | 提纯 — 这对我没用 |

大纲的优势在于：**Think 和 Connect 用同一个界面**。你在日记里看到昨天写的想法，把它拖到一个主题节点下，加上标签，链接到一条旧笔记——这个过程看起来只是"整理"，实际上你在发现自己思想的结构。

#### Connect 的行为准则

- **标签是索引，不是分类** — 标签帮助未来找到笔记，而不是建立完美的知识体系。过度打标签不如没有标签
- **标签从简单开始** — 不要设计完美体系，让标签随使用自然生长
- **寻找关系，而非追求整齐** — Connect 的价值在于发现你没意识到的联系，不在于笔记看起来有多整齐

### Compound — 积累的思考产生复利

> "知识的积累就像滚雪球一样，你已有的知识好比一颗雪球，当它在雪地上滚动时，能够吸附的雪就是你新获得的知识，你原本的雪球越大，可能吸附的雪也就越多。"

这是所有笔记工具的死亡之谷：记了 → 沉底 → 再也没看到 → 觉得白记 → 不记了。

Compound 是让循环转起来的关键：你积累的、连接过的思考，在未来产生超过各部分之和的价值。每一条新笔记让所有旧笔记更有价值，就像复利——时间越长，回报越大。

Compound 可以通过多种方式发生，这是一个我们持续探索的开放方向：

| 方式 | 描述 | 用户感受 |
|------|------|---------|
| **浏览器上下文** | 打开与笔记相关的网页，旧笔记主动浮现 | "我之前想过这个" |
| **AI 照亮模式** | AI 发现你笔记中隐藏的矛盾、共识、转折点 | "原来我是这么想的" |
| **AI 对话找方法** | 基于你的笔记回答具体问题 | "我之前的思考能帮上忙" |
| **直接执行** | 笔记中的方法论变成可执行的行动 | "我可以直接用这个" |
| **定期 Review** | AI 辅助回顾，发现阶段性的思考模式 | "这段时间我一直在想这个" |

AI 在 Compound 中扮演关键角色，但方向是增强而非替代：

| AI 层次 | AI 做的事 | 用户感受 |
|---------|----------|---------|
| **检索** | 找到相关的旧笔记 | "还好没丢" |
| **总结** | 压缩和格式化外部信息 | "工具很强" |
| **连接** | 在笔记之间发现关联 | "原来这两件事有关系" |
| **洞察** | 发现用户自己没意识到的模式 | **"原来我是这么想的"** |

前两层是工具能力，后两层才是认知增强。soma 的 AI 应该朝着第四层发力——不生成新内容，而是**把用户散落的想法照亮**。AI 不替你思考，而是把你自己的思考照亮。

在 AI 的加持下，Compound 不是线性增长，而是指数级的——AI 能在你所有笔记之间发现你自己看不到的模式、矛盾和关联。

## 三、正反馈循环

> "付出小于回报时，人们就会坚持；付出大于回报时，人们就会放弃。" — 岩田聪

三步循环要转起来，**每一步都需要有内在奖励**，而不是等到最后才有回报：

| 步骤 | 内在奖励 |
|------|---------|
| **Think** | 用自己的话写出来的那一刻，你就理解了。理解本身就是奖励 |
| **Connect** | 发现两条笔记之间的关系时，那就是一个小洞察。发现本身就是奖励 |
| **Compound** | 过去的思考帮你解决了当下的问题。有用本身就是奖励 |

正反馈循环的三个条件：

1. **Think 的付出足够低** — soma 就在侧边栏，高亮一下、写一句话，3 秒完成
2. **Compound 来得足够快** — 不是三个月后回顾时才发现有用，而是下次浏览相关网页时就出现
3. **Compound 感受足够强** — 不是"这条笔记还在"，而是"这条笔记帮我解决了当下的问题"

### 冷启动问题

新用户刚开始没有笔记，没有笔记就没有 Compound，没有 Compound 就感受不到价值。因此新用户引导的核心目标不是教功能，而是**尽快让用户完成第一次"想了 → 写了 → 用上了"的体验**。

## 四、soma 的独特位置

### 为什么是浏览器侧边栏？

三个维度上，soma 做到了独立应用做不到的事：

| 维度 | 独立应用的痛点 | soma 的浏览器优势 |
|------|--------------|------------------|
| **Think** | 需要切换应用，打断心流 | 在阅读的地方思考，笔记和网页并排显示 |
| **Connect** | 记录时已脱离原始语境 | 上下文始终在旁边，浏览器知道你正在看什么 |
| **Compound** | 需要用户主动打开工具回顾 | 浏览网页时自动触发，笔记主动找到你 |

### 与同类产品的关系

soma 的方法论吸收了四款产品的核心理念，但通过浏览器原生位置实现了它们做不到的闭环：

- **flomo 的"低摩擦记录"** → soma 的 Think 比 flomo 更自然（在阅读的地方思考，不需要切换应用）
- **Tana 的"渐进式结构化"** → soma 用 Supertag + 字段实现 Connect（先记后结构）
- **Roam 的"意外发现"** → 上下文感知侧边栏就是浏览器版的反向链接面板
- **Heptabase 的"理解优先"** → 大纲中的 Connect 操作就是另一种形式的空间思维

## 五、设计原则

### 关于 Think

1. **好的工具帮助用户更加专注，坏的工具分散注意力** — 每个功能都要问：这是帮用户专注于思考，还是在抢夺注意力？
2. **用自己的话写，才算真正理解** — 费曼学习法的实践：笔记是把知识讲给未来的自己听
3. **少即是多，不是越少越好，而是增加信息密度** — 界面上的每个元素都应该传递有用信息，减少装饰和噪音

### 关于 Connect

4. **标签是索引，不是分类** — 标签帮助未来找到笔记，而不是建立完美的知识体系。过度打标签不如没有标签
5. **删除是日常动作** — 如果真心舍不得某条内容，你会有写点东西、把核心价值保留下来的冲动。认同和触动不等于有用
6. **工具即过程** — 通过工具的使用流程，帮助用户塑造思考和连接的习惯，而不只是提供功能

### 关于 Compound

7. **教育用户最好的时机是帮助他们实现正在积极努力的目标** — 笔记出现的最佳时机是用户正在处理相关问题的时候
8. **AI 帮你发现和使用你的思考，而不是替你思考** — AI 是照亮和连接的引擎，不是内容生成的替代品

### 关于产品

9. **正反馈是成功的关键，负反馈带来的更多是消耗** — 把记录从"任务"变成"收获"，关注"我今天又理解了 Y"而不是"我还没学会 X"
10. **接受不完美，Demo 先行** — 不要对着空泛的文本讨论，先做出来，基于实物迭代。迭代思维，不要总是重新开始
11. **如果工具所解决的问题并不存在，那么这个工具就不可能是好工具** — 每个功能都要回答：用户在没有这个功能时，真的痛吗？

## 六、soma 不做什么

| 不做 | 原因 |
|------|------|
| **不做 All-in-One** | 功能堆砌分散注意力，违背"少即是多" |
| **不做自动整理** | AI 替你整理 = 你跳过了 Think 和 Connect，笔记依然是别人的知识 |
| **不做知识图谱可视化** | 看起来强大但使用门槛高，大多数人用不上 |
| **不做社交/分享** | 笔记是写给自己的，为别人写会扭曲记录的动机 |
| **不追求笔记数量** | 100 条思考过的笔记，胜过 10000 条未处理的剪藏 |

## 七、一句话总结

> **Think where you read.**

soma 让你在阅读的时候思考，在组织的时候发现脉络，在积累中产生复利。这是一个浏览器原生的知识循环：Think 靠侧边栏的零摩擦，Connect 靠大纲的认知操作，Compound 靠 AI 的指数级加持。三步闭环，正反馈驱动。

---

*"真正重要的不是未来会改变什么，而是什么永远不会改变。" — 贝佐斯*

*人们永远需要记录想法，永远需要在需要的时候找到它们。soma 围绕这两件不会改变的事来构建。*

## 八、开放问题

> 尚未决定的产品方向，记录思考过程，想清楚后落实到 TASKS.md。

### 剪藏的边界 ✅ 已决定

**结论**：用户侧只创建 `#source` 元数据节点（URL、标题、来源），不存也不展示正文。AI 需要上下文时通过 content script 按需抓取（On-Demand Fetch）。

**决策依据**（详细调研见 2026-03-06 研究）：
- 竞品规律：保存全文的是"阅读环境"（Readwise、Evernote），不保存的是"思考工具"（flomo、Hypothesis）。soma 是思考工具
- 收集者谬误：Pocket 数据显示每存 3 篇只有 1 篇被打开；书签回访率不到 20%。保存能力越强，囤积越严重
- 浏览器扩展优势：soma 有 content script，随时按需获取——不存正文是选择，不是限制
- 实体型剪藏（书影音）同理：只存元数据字段，不存内容
- 未来可渐进升级到 Shadow Cache（IndexedDB + TTL 自动过期），但起步不需要

已落实到 TASKS.md。

### 上下文感知 Sidebar：信息气味指示器

**结论**：不做纯 Push（主动弹出内容），也不做纯 Pull（完全依赖用户记忆）。采用"信息气味指示器"——三层渐进式披露，每一步都是用户主动选择。

**设计方案**（详细调研见 2026-03-06 研究）：

| 层 | 形态 | 触发 | 认知成本 |
|---|------|------|---------|
| L1 | 工具栏图标旁的低调数字 badge（如 `3`） | 自动后台匹配 | 接近零 |
| L2 | 点击 badge → 相关笔记标题列表（最多 5 条） | 用户点击 | 低 |
| L3 | 点击某条 → 导航到该节点 | 用户点击 | 正常 |

**理论基础**：
- 信息觅食理论（Pirolli & Card）：提供"信息气味"让用户自己决定是否跟踪，不替用户做决定
- 宁静技术原则（Amber Case）：信息在注意力外围告知，不在中心打断
- 注意力研究：被打断后需要 23 分恢复专注（Gloria Mark），badge 不算打断因为不要求行动

**关键约束**：
- badge 用低饱和度颜色（Sage Green），无动画，无红色——避免 badge 焦虑
- 精确率 > 召回率：宁可漏掉也不推错。一次误判 = 用户永久关闭功能
- 从 URL 精确匹配起步（零误判），再扩展到关键词/语义匹配
- 笔记太少时不激活此功能

**实施节奏**：v1 URL 精确匹配 → v2 关键词/语义匹配 → v3 可选"共读模式"（远期）

### AI 照亮的时机：Spark Review → 反思对话 → Thinking Pulse

**结论**：AI 照亮是用户主动发起的独立动作，不是实时伴随。三阶段渐进。

**认知科学基础**（详细调研见 2026-03-06 研究）：
- **生成效应**（Slamecka & Graf）：自己发现的洞察比 AI 告知的记忆更深、理解更好。AI 应创造发现条件，而非递送结果
- **反思最佳时机**（Pennebaker）：写完后 3-7 天——有足够心理距离产生新视角，又还记得足够细节
- **解释水平理论**（Trope & Liberman）：时间距离越远，思维越抽象，越容易看到模式而非细节

**三阶段方案**：

**Phase 1 — Spark Review（近期）**：AI 精选 3-5 组"火花"，每组 2-3 条有隐藏关联的旧笔记。并置呈现为主（最大化生成效应）+ 一句极简提示为辅。不是每天推送，而是"有足够新材料时"才生成。

**Phase 2 — AI 反思对话（中期）**：用户从 Spark 发现线索 → 启动苏格拉底式对话。AI 不给结论，通过提问引导发现。对 AI 质量要求极高。

**Phase 3 — Thinking Pulse（远期）**：月度"思考脉冲"报告，发现跨时间、跨主题的深层模式。需要 3+ 个月活跃使用数据。

**设计红线**（所有阶段适用）：
1. 永远不在用户写作时弹 AI 建议
2. AI 展示的永远是用户自己写的内容，不生成新观点
3. 用户可以反馈"这个关联没意义"
4. 笔记不够多时坦诚说"再积累一些"

**竞品机会**：没有产品真正做到"洞察"层。Readwise Daily Review 最接近但靠间隔重复让用户自己发现，不是 AI 主动揭示。这是 soma 的空白地带。
