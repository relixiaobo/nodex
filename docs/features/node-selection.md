# Feature: 节点选中

> Phase 1 | 单选已实现，多选及批量操作未实现

## 概述

节点交互存在两种互斥模式：**编辑模式**（光标闪烁，正在打字）和**选中模式**（节点高亮，无光标，可执行整体操作）。同一时间只能处于其中一种，或都不处于（空闲状态）。

多选允许用户同时操作多个节点（删除、缩进、移动等），是高效编辑的基础。

### 互斥规则

| 操作 | focusedNodeId | selectedNodeId(s) |
|------|--------------|-------------------|
| 点击普通节点文字 | 设为该节点 | 清空 |
| 点击引用节点 | 清空 | 设为该节点 |
| 双击引用节点 | 设为该节点 | 清空 |
| Escape（编辑中） | 清空 | 设为当前节点 |
| Escape（选中中） | 不变 | 清空 |
| Enter / 打字（选中中） | 设为第一个选中节点 | 清空 |

## 行为规格

### 两种交互模式

| 模式 | 视觉 | 用户体验 |
|------|------|---------|
| 编辑模式 | 光标闪烁在节点内 | 正在打字，NodeEditor 已挂载 |
| 选中模式 | ring 高亮（无光标） | 节点被选中，可执行批量操作 |

### 进入与退出

| 用户操作 | 效果 |
|---------|------|
| 点击节点文字 | 进入编辑模式，光标出现在点击位置 |
| 点击节点非文字区域 | 进入编辑模式，光标出现在文末 |
| Esc（编辑中） | 退出编辑，当前节点变为选中状态 |
| Esc（选中中） | 取消所有选中，回到空闲状态 |
| Enter 或打字（选中中） | 回到编辑模式，聚焦第一个选中节点 |
| ↑/↓（选中中） | 回到编辑模式，聚焦选区边缘节点的邻居 |

### 单选

- 点击任意节点 → 该节点被选中并进入编辑模式
- 如果之前有多个节点被选中，先全部取消
- 引用节点例外：单击 → 选中（不编辑），双击 → 编辑

### 多选 — 未实现

四种方式进入多选：

#### Cmd/Ctrl+Click（逐个切换）
- 点击未选中的节点 → 加入选中集合
- 点击已选中的节点 → 从集合中移除
- 多选时自动退出编辑模式

#### Shift+Click（范围选中）
- 以上一次选中的节点为锚点（`selectionAnchorId`）
- 到 Shift+Click 的目标节点为终点
- 选中锚点与终点之间所有可见节点（使用 `flattenVisibleNodes` 计算范围）
- 自动退出编辑模式

#### 鼠标拖拽（框选）
- 在非文字区域按住鼠标拖动 → 框内节点被选中
- 在文字区域开始拖动 → 如鼠标停留在同一节点内，正常选择文字；越过到其他节点时切换为节点框选
- 仅左键、无修饰键时触发

#### Shift+Arrow ↑/↓（键盘扩展选区）
- 编辑模式中按 Shift+↑/↓：第一下选中当前节点并退出编辑，后续每下向该方向扩展一个节点
- 选中模式中按 Shift+↑/↓：持续扩展选区

#### Cmd+A（全选）
- 编辑模式中：选中当前节点内所有文字（TipTap 标准行为）
- 选中模式中：选中当前视图下所有顶层节点

### 子树整体选中

- 选中一个节点 = 选中它和它的所有后代
- 无需逐个选中后代节点
- 先选中子节点再选中父节点 → 子节点单独选中被合并，只保留父节点
- 已被父节点覆盖的后代无法单独取消选中

### 选中模式下的批量操作 — 未实现

| 快捷键 | 操作 |
|--------|------|
| Backspace / Delete | 删除所有选中节点 |
| Tab | 缩进所有选中节点 |
| Shift+Tab | 反缩进所有选中节点 |
| Cmd+Shift+D | 复制所有选中节点及子树 |
| Cmd+Enter | 批量切换完成状态 |

所有批量操作支持撤销（依赖 undo-redo 系统）。

### 选中自动清除的时机

- 普通点击任意节点时（切换为单选/编辑）
- 按 Esc 时
- Zoom 导航切换时（进入/退出节点聚焦视图）
- 部分批量操作完成后（如复制）

### 引用节点的特殊行为

| 操作 | 普通节点 | 引用节点 |
|------|---------|---------|
| 单击 | 进入编辑模式 | 仅选中（ring 高亮） |
| 双击 | 选中文字 | 进入编辑模式 |
| 选中后按 → | 无特殊效果 | 将节点引用转换为行内引用 |

### 选中 ring 样式

当前实现（OutlinerItem）：
- Selection ring 包裹 `BulletChevron` + 文本内容区域
- 不包裹 ChevronButton（展开/折叠按钮）
- 样式：`ring-1 ring-primary/40 rounded-sm bg-primary/5`
- 多选时每个选中节点独立显示 ring

## 实现考量

### Escape 键优先级

当前 Escape 仅关闭下拉菜单（#tag / @reference 选择器）。需要扩展为：

1. 如果有下拉菜单打开 → 关闭菜单（现有行为）
2. 如果在编辑模式（无下拉菜单）→ 退出编辑，当前节点变为选中
3. 如果在选中模式 → 取消所有选中

### 多选与 FieldValueOutliner 的关系

- 多选仅在主 OutlinerView 中有效
- FieldValueOutliner 内的节点不参与多选
- 在 FieldValueOutliner 上下文中，Shift+Arrow 应触发 `onNavigateOut` 而非扩展选区

### 可见节点范围计算

Shift+Click 范围选中需要计算两个节点之间的可见节点列表。使用 `tree-utils.ts` 中的 `flattenVisibleNodes` 工具函数，结合 `expandedNodes` 状态确定哪些子树是展开的。
