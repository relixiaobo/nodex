# Chat Drawer 详细设计

> Outliner 为底，Chat 为抽屉。笔记是焦点，AI 是环境。
>
> **2026-03-23** — 方案 D 细化

---

## 三种状态

整个界面只有三种状态，状态之间的转换通过输入框和手势完成：

### 状态 1：Outliner 全屏（默认）

```
┌──────────────────────────────┐
│  W / ... / Week 12           │  ← 面包屑
│  # day                  ···  │
│  Today, Sun, Mar 23          │
│  < Today >  | 📅             │
│                              │
│  • 节点 1                     │
│  • 节点 2                     │
│    • 子节点                   │
│  • 节点 3                     │
│                              │
│                              │
│  ┌──────────────────────  👤│
│  │ 💬 Ask about your notes..││  ← 悬浮输入栏
│  └──────────────────────────│
└──────────────────────────────┘
```

用户 80% 时间在此状态：浏览和编辑笔记。输入栏安静地待在底部，不干扰。

### 状态 2：Chat 抽屉打开

```
┌──────────────────────────────┐
│  W / ... / Week 12           │  ← Outliner 顶部露出（可点击回到状态 1）
│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│  ← 半透明遮罩
│┌────────────────────────────┐│
││ ── 拖拽条 ──                ││  ← 下拉关闭
││ 💬 会话标题       ⟳ ✕      ││  ← 新建对话 / 关闭
││                            ││
││ 🧑 你: 帮我分析这个...       ││
││                            ││
││ 🤖 AI: 分析如下...          ││
││                            ││
││  ┌────────────────────────┐││
││  │ 继续对话...             │││  ← 输入框在抽屉底部
││  └────────────────────────┘││
│└────────────────────────────┘│
└──────────────────────────────┘
```

用户发送消息后进入此状态。抽屉从底部滑出，占据约 75% 高度。

### 状态 3：Chat 全屏

```
┌──────────────────────────────┐
│┌────────────────────────────┐│
││ ── 拖拽条 ──                ││  ← 下拉恢复到状态 2
││ 💬 会话标题       ⟳ ✕      ││
││                            ││
││ 🧑 你: ...                  ││
││                            ││
││ 🤖 AI: ...                 ││
││  (更多对话内容)              ││
││                            ││
││  ┌────────────────────────┐││
││  │ 继续对话...             │││
││  └────────────────────────┘││
│└────────────────────────────┘│
└──────────────────────────────┘
```

用户将抽屉拖拽到顶部时进入全屏。适合深度对话。下拉可恢复到半高。

## 悬浮输入栏

### 视觉

```
┌──────────────────────────  👤│
│ 💬  Ask about your notes...  ││
└──────────────────────────────│
```

- 位置：Outliner 底部，悬浮在内容之上（`position: sticky` 或 `absolute bottom`）
- 宽度：与内容卡片等宽，底部圆角与卡片对齐
- 高度：约 44px（单行输入 + padding）
- 左侧：💬 Chat 图标（点击可打开上一次对话，不发送新消息）
- 右侧：👤 用户头像（与当前顶部栏的头像功能一致，移到此处）
- 背景：`bg-background` + 顶部模糊边缘（`backdrop-blur` 或渐变遮罩），让下方 Outliner 内容不干扰
- 样式：安静、不抢眼，像 macOS Dock 那样"在那里但不打扰"

### 交互

| 操作 | 行为 |
|------|------|
| 点击输入框 | 聚焦，可输入文字 |
| 输入文字 + Enter | 发送消息 → 打开抽屉（状态 2） |
| 点击 💬 图标 | 打开上一次对话的抽屉（不发送新消息） |
| 点击 👤 头像 | 打开用户菜单（同现在的 ToolbarUserMenu） |
| `Cmd+L` | 聚焦输入框 |

### 输入框是否展开？

**不展开**。保持单行。理由：
- Side Panel 宽度有限，展开多行 + 模型选择器会遮挡大量 Outliner 内容
- 发送第一条消息后抽屉打开，后续对话在抽屉内的完整 ChatInput 中进行
- 这个输入框的定位是"快速入口"，不是"完整编辑器"

### 模型选择

- 不在悬浮输入栏显示模型选择器（太拥挤）
- 模型选择在抽屉打开后的 ChatInput 中完成
- 或者：长按发送按钮显示模型选择快捷菜单

## 抽屉行为

### 打开

- **触发**：发送消息 / 点击 💬 图标 / `Cmd+L`（无输入时）/ 点击 Chat 历史
- **动画**：从底部滑出，250ms `ease-out`
- **默认高度**：75% 视口高度
- **Outliner 露出**：顶部约 40-60px（面包屑可见）

### 关闭

- **下拉拖拽**：拖拽条下拉超过阈值（30%）→ 关闭
- **点击遮罩**：点击 Outliner 露出的面包屑区域 → 关闭
- **关闭按钮**：抽屉头部的 ✕ → 关闭
- **`Escape`**：关闭抽屉
- **动画**：滑回底部，200ms `ease-in`

### 高度调节

- **拖拽条**：抽屉顶部有一个拖拽指示器（短横线），可上下拖拽
- **三个吸附点**：
  - **半高**：75% 视口高度（默认）
  - **全屏**：100%（拖到顶部吸附）
  - **关闭**：下拉超过阈值自动关闭
- **记住偏好**：用户手动调整的高度在会话内保持

### 抽屉内结构

```
┌────────────────────────────┐
│ ━━━━ (拖拽条)               │  4px，居中
│ 💬 会话标题    ⟳  ✕         │  36px header
├────────────────────────────┤
│                            │
│  对话消息列表               │  flex-1, scroll
│  (复用 ChatPanel)           │
│                            │
├────────────────────────────┤
│  ChatInput (完整版)          │  模型选择器 + 多行输入
└────────────────────────────┘
```

- **拖拽条**：视觉指示器 + touch/mouse 拖拽区域
- **Header**：会话标题（可编辑）+ 新建对话 ⟳ + 关闭 ✕
- **消息区**：直接复用 ChatPanel 的消息渲染（去掉 header，因为抽屉自己有）
- **输入区**：复用 ChatInput，完整功能（多行、模型选择、发送）

## Outliner 顶部

### 去掉标签栏

方案 D 不需要 Chat/Node toggle 标签栏。顶部恢复为：

```
┌──────────────────────────────┐
│  W / ... / Week 12      ··· │  ← 面包屑 + 更多菜单
├──────────────────────────────┤
│  内容区域                     │
```

或者更简洁：面包屑直接在内容卡片内部（当前已有），不需要额外的顶部栏。

### 用户头像

从顶部标签栏移到悬浮输入栏右侧。理由：
- 标签栏去掉了，头像需要新位置
- 输入栏是常驻元素，头像跟着它始终可见
- 位置合理——输入栏是交互热区，用户菜单在这里触手可及

## Chat 历史

### 入口

1. **💬 图标点击**：打开上一次对话（最常用）
2. **⌘K 命令面板**：搜索历史对话
3. **抽屉内 ⟳ 按钮**：新建对话
4. **抽屉内标题点击**：下拉显示最近对话列表（可选）

### 会话切换

- 在抽屉打开时，点击 ⟳ 新建 → 清空对话区，开始新会话
- 在 ⌘K 中选择历史对话 → 抽屉内切换到该会话
- 旧会话自动保存

## 节点引用交互

### Chat 中点击 `<ref>`

1. 关闭抽屉（动画滑回底部）
2. Outliner 导航到该节点
3. 悬浮输入栏恢复

用户可以查看节点内容，然后点击 💬 或输入新问题继续对话。

### Chat 中的 `<node />` 内嵌

- 在抽屉内正常渲染（内嵌 outliner）
- "Open in outliner" 操作 → 关闭抽屉，Outliner 导航到该节点

### Outliner 中的 @ 引用

- Outliner 输入时 `@` 触发节点选择器（现有功能不变）
- 悬浮输入栏中也支持 `@` 引用（注入上下文给 AI）

## 快捷键

| 快捷键 | 行为 |
|--------|------|
| `Cmd+L` | 聚焦悬浮输入框（抽屉关闭时）/ 聚焦抽屉内输入框（抽屉打开时）|
| `Escape` | 关闭抽屉 → 回到 Outliner |
| `Cmd+Shift+D` | Outliner 导航到 Today |
| `Cmd+K` | 命令面板（搜索节点 / Chat 历史）|

## 状态管理（ui-store）

```typescript
// 替换 Toggle 模型
interface UIStore {
  // Outliner（始终存在）
  currentNodeId: string | null;
  nodeHistory: string[];
  nodeHistoryIndex: number;

  // Chat 抽屉
  chatDrawerOpen: boolean;
  chatDrawerHeight: number;          // 0-1 比例，默认 0.75
  currentChatSessionId: string | null;

  // 操作
  openChatDrawer(sessionId?: string): void;   // 打开抽屉
  closeChatDrawer(): void;                     // 关闭抽屉
  setChatDrawerHeight(height: number): void;   // 拖拽调整高度
  navigateToNode(nodeId: string): void;        // Outliner 导航
  goBackNode(): void;
  goForwardNode(): void;

  // 移除
  // activeView — 不再需要（Outliner 始终活跃）
  // switchToChat / switchToNode — 不再需要
}
```

### 持久化

- `currentNodeId`、`currentChatSessionId` → chrome.storage
- `chatDrawerOpen` → 不持久化（每次打开默认关闭）
- `chatDrawerHeight` → chrome.storage（记住偏好）

## 组件架构

```
App.tsx
  └── DrawerLayout.tsx (新)
        ├── OutlinerView (始终可见)
        │     ├── Breadcrumb
        │     ├── NodeHeader
        │     ├── OutlinerView / FieldList
        │     └── FloatingChatBar (新, sticky bottom)
        │           ├── 💬 图标 (打开上次对话)
        │           ├── 输入框 (发送 → 打开抽屉)
        │           └── 👤 头像
        │
        └── ChatDrawer (条件渲染, 覆盖层)
              ├── DragHandle
              ├── DrawerHeader (标题 + 新建 + 关闭)
              ├── ChatPanel (复用, hideHeader=true)
              └── ChatInput (复用)
```

### 保留不变的组件

- `ChatPanel.tsx` — 消息渲染，不含布局假设
- `NodePanel.tsx` — 节点内容，不含布局假设
- `OutlinerView.tsx` — 大纲渲染
- `ChatInput.tsx` — 完整输入区域
- `CommandPalette.tsx` — 命令面板

### 新建的组件

- `DrawerLayout.tsx` — 替换 ToggleLayout，管理 Outliner + 抽屉
- `FloatingChatBar.tsx` — 底部悬浮输入栏
- `ChatDrawer.tsx` — 抽屉容器（拖拽 + 遮罩 + 动画）

### 删除的组件

- `ToggleLayout.tsx` — 被 DrawerLayout 替换

## 实施计划

### Phase 1：基础结构

## 设计红线

1. **Outliner 始终可见** — 抽屉关闭时是全屏 Outliner，抽屉打开时顶部仍露出面包屑
2. **输入即对话** — 不需要先"打开 Chat"，在输入框打字+回车就开始
3. **不丢状态** — 关闭抽屉再打开，对话仍在；切换节点再回来，展开状态仍在
4. **不加层级** — 只有 Outliner + 抽屉两层，不出现抽屉里的抽屉、弹窗里的弹窗
5. **输入栏不膨胀** — 悬浮输入栏始终单行，不展开为多行编辑器
