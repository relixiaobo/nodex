# Feature: 节点选中

> 全量规格 | 基于 Tana 双模式选择范式

## 概述

大纲编辑器中存在两种互斥的交互模式：

| 模式 | 用户所见 | 含义 |
|------|---------|------|
| **编辑模式** | 某个节点内有闪烁的文本光标 | 用户正在编辑该节点的文字内容 |
| **选中模式** | 一个或多个节点整行高亮，无文本光标 | 用户正在对节点整体进行操作 |
| **无选择** | 无任何高亮或光标 | 空闲状态 |

多选允许用户同时操作多个节点（删除、缩进、移动等），是高效编辑的基础。

## 一、模式切换

```
                    ┌──────────────────┐
                    │    无选择状态      │
                    └────────┬─────────┘
                             │ 点击节点
                             ▼
                    ┌──────────────────┐
          ┌────────│    编辑模式        │────────┐
          │        │ (光标在节点内)      │        │
          │        └──────────────────┘        │
          │              │                      │
          │   Esc        │ Shift+↑/↓            │ Shift+↑/↓
          │              │ (首次按键)            │ (后续按键)
          │              ▼                      │
          │        ┌──────────────────┐        │
          │        │    选中模式        │◄───────┘
          │        │ (整行高亮,无光标)   │
          │        └──────────────────┘
          │              │        │
          │   Enter/     │        │ Esc
          │   输入字符    │        │
          │              │        ▼
          │              │  ┌──────────────────┐
          └──────────────┘  │    无选择状态      │
                            └──────────────────┘
```

### 编辑模式 → 选中模式

| 触发 | 行为 |
|------|------|
| **Esc**（无下拉菜单打开） | 光标消失，当前节点保持高亮，编辑器变为只读 |
| **Shift+↑/↓**（首次） | 清除浏览器文本选区，退出编辑，当前节点成为选择锚点 |

### 选中模式 → 编辑模式

| 触发 | 行为 |
|------|------|
| **↑**（无修饰键） | 清除选区，编辑选区上方相邻可见节点，光标在**文本末尾** |
| **↓**（无修饰键） | 清除选区，编辑选区下方相邻可见节点，光标在**文本开头** |
| **Enter** | 聚焦第一个选中节点，光标在文本末尾 |
| **输入可打印字符** | 聚焦第一个选中节点，字符追加到文本末尾 |

> 光标位置约定：↑ 到末尾、↓ 到开头，符合用户从上/下方进入节点的直觉。

### 选中模式 → 无选择

| 触发 | 行为 |
|------|------|
| **Esc** | 清除所有选择 |

### 无选择 → 编辑模式

| 触发 | 行为 |
|------|------|
| **点击任意节点** | 聚焦该节点，光标由浏览器根据点击坐标放置 |

### Esc 三级退出链

连续按 Esc 依次经过：**编辑模式 → 选中模式 → 无选择**。如果当前处于 Zoom（聚焦某节点的子树视图），无选择状态下按 Esc 还会退出 Zoom（PanelStack.pop）。

### Escape 键优先级

1. 如果有下拉菜单打开（#tag / @reference） → 关闭菜单（现有行为）
2. 如果在编辑模式（无下拉菜单）→ 退出编辑，当前节点变为选中
3. 如果在选中模式 → 取消所有选中

## 二、鼠标交互

### 2.1 普通点击

1. 如果刚结束拖选操作，忽略这次点击（防止 mouseup 干扰）
2. 如果存在多选（≥2 节点），先清除多选
3. 将点击节点设为唯一选中项
4. **点击文本区域**：进入编辑模式，光标位置由浏览器根据点击坐标自然放置
5. **点击非文本区域**（左侧 padding、bullet 区域）：进入编辑模式，光标放在文本末尾

### 2.2 Cmd+Click（多选切换）

- 节点已选中 → 取消选中
- 节点未选中 → 加入选中集合
- 选中结果 ≥ 2 个节点时，自动退出编辑模式进入选中模式
- **子树合并吸收规则**：
  - 点击已选父节点的后代 → **忽略**（已被祖先覆盖）
  - 点击已选后代节点的祖先 → 移除所有已选的后代，只保留祖先（合并吸收）

### 2.3 Shift+Click（范围选择）

- 从**锚点节点**到点击节点之间的所有**可见节点**全部选中
- 可见节点 = 未被折叠隐藏的节点，按深度优先遍历排列
- 仅保留根级选中：如果父子节点都在范围内，只保留父节点
- 退出编辑模式，进入选中模式

### 2.4 双击

- 普通节点：浏览器默认行为（选中单词）
- 引用节点：进入编辑模式，光标在文本末尾（单击引用节点不进入编辑，双击才会）

## 三、拖动选择（Drag Select）

### 启动条件

- 左键按下，无 Cmd/Shift/Ctrl 修饰键
- 鼠标垂直移动超过 **5px** 阈值

### 文本区域 vs 非文本区域

- **从文本区域开始拖动**：只有当鼠标移动到**另一个**节点行时才激活拖选。在同一节点内拖动视为浏览器原生文本选择
- **从非文本区域开始拖动**：超过 5px 阈值即激活拖选

> 设计意图：在编辑器上拖动大概率是选择文字，只有跨节点才说明意图是选择整行节点。

### 拖选过程

1. 清除浏览器文本选区
2. 起始节点被选中并设为锚点
3. 退出编辑模式，进入选中模式
4. 鼠标移动过程中，实时更新选区：选中从锚点到当前悬停节点之间的所有可见节点
5. 添加 `select-none` 样式防止文本选择伪影

### 拖选结束

- 释放鼠标后，选区保持
- 抑制随后的 mouseup 点击事件（下一个事件循环重置），避免点击干扰刚完成的选区

### 实现要点

- 必须使用 **document 级别**的 mousemove/mouseup 监听。原因：contenteditable 元素会捕获鼠标事件，导致父容器上的 React onMouseMove 不可靠触发

## 四、键盘快捷键

### 4.1 编辑模式下

| 按键 | 行为 |
|------|------|
| **Esc**（无下拉菜单） | 退出编辑 → 进入选中模式（当前节点保持高亮） |
| **↑** | 光标移到上一可见节点，光标在**文本末尾** |
| **↓** | 光标移到下一可见节点，光标在**文本开头** |
| **Shift+↑/↓**（首次） | 退出编辑进入选中模式，当前节点为锚点 |

### 4.2 选中模式下

| 按键 | 行为 |
|------|------|
| **Esc** | 清除所有选择 → 无选择状态 |
| **↑** | 退出选中 → 编辑选区上方相邻节点（光标在末尾） |
| **↓** | 退出选中 → 编辑选区下方相邻节点（光标在开头） |
| **Shift+↑** | 从锚点向上扩展选区一个节点 |
| **Shift+↓** | 从锚点向下扩展选区一个节点 |
| **Enter** | 编辑第一个选中节点，光标在末尾 |
| **可打印字符** | 编辑第一个选中节点，字符追加到末尾 |
| **Cmd+A** | 选中当前根节点的所有直接子节点 |
| **Backspace / Delete** | 批量删除所有选中节点，焦点移到最上方被删节点的上一个节点 |
| **Tab** | 批量缩进所有选中节点 |
| **Shift+Tab** | 批量取消缩进 |
| **Cmd+Shift+D** | 批量复制所有选中节点，完成后清除选区 |
| **Cmd+Enter** | 批量切换 Checkbox（有任何未完成 → 全部完成；全部已完成 → 全部取消） |

## 五、子树选择

### 核心原则

**选中父节点 = 隐式选中所有后代。** 选择集合只存储根级选中节点，不存储被祖先覆盖的后代。

### 判断节点是否被选中

沿所有权链（parent chain）向上遍历，只要任一祖先被选中，该节点即为"被选中"状态。

### 对各操作的影响

| 操作 | 处理逻辑 |
|------|---------|
| Cmd+Click 后代 | 后代已被祖先覆盖，忽略操作 |
| Cmd+Click 祖先 | 移除已选的后代，只保留祖先（合并吸收） |
| 范围选择 | 计算完范围后过滤：父子都在范围内时只保留父 |
| Shift+Arrow 扩展 | 同样的合并吸收逻辑 |
| 批量删除 | 只删除根级选中节点（后代随父节点一同删除） |
| 批量缩进/取消缩进 | 只操作根级选中节点 |

## 六、选中态视觉呈现

### 6.1 普通节点 — 双层高亮

1. **子树遮罩层**：浅蓝色半透明背景，覆盖被选中节点的**整个子树区域**（节点行 + 所有展开的子节点），圆角。层级在内容下方，不影响交互
2. **当前节点行高亮**：较深的蓝色背景，仅覆盖被选中的根节点行（bullet + 文本区域）

**视觉效果**：
- 单选一个有子节点的父节点：用户看到整个子树区域被浅蓝色包裹，父节点行更深蓝
- 多选多个节点：每个根选中节点各自一块子树遮罩，互相独立
- 被隐式选中的子节点：不显示独立高亮，被父节点的浅蓝遮罩覆盖

### 6.2 引用节点

- 不使用全行遮罩
- 使用**紧凑蓝色边框**包裹文本内容
- 边框宽度收缩到实际文本宽度（不撑满整行）

### 6.3 暗色模式

- 子树遮罩：深蓝半透明（20% 不透明度）
- 节点行高亮：深蓝半透明（40% 不透明度）

### 6.4 当前实现（过渡方案）

- Selection ring 包裹 `BulletChevron` + 文本内容区域
- 不包裹 ChevronButton（展开/折叠按钮）
- 样式：`ring-1 ring-primary/40 rounded-sm bg-primary/5`
- 多选时每个选中节点独立显示 ring
- 后续迁移到双层高亮方案

## 七、引用节点的特殊选择行为

| 交互 | 普通节点 | 引用节点 |
|------|---------|---------|
| **单击** | 进入编辑模式，出现文本光标 | 仅选中（只读，紧凑蓝色边框），不进入编辑 |
| **双击** | 浏览器默认选中单词 | 进入编辑模式，光标在目标节点文本末尾 |
| **Enter（选中态）** | 编辑当前节点，光标在末尾 | 编辑引用节点，光标在目标文本末尾 |
| **→（选中态，单个引用被选中）** | 无特殊行为 | 将节点引用转换为内联引用 |
| **Backspace（光标在开头，有文字）** | 与上一节点合并文本 | 导航到上一节点（不合并，引用不支持文本合并） |
| **Backspace（空内容）** | 删除节点 | 删除引用节点 |
| **高亮样式** | 全行浅蓝遮罩 + 深蓝行高亮 | 紧凑蓝色边框（宽度收缩到文本） |

## 八、锚点机制

"锚点"是范围选择和 Shift+Arrow 的参考起点：

| 触发 | 锚点设为 |
|------|---------|
| 普通点击节点 | 点击的节点 |
| Shift+↑/↓ 首次按键 | 当前编辑中的节点 |
| 拖选开始 | 拖选起始节点 |

- Shift+Click 的范围 = 从锚点到点击目标之间的所有可见节点
- Shift+Arrow 的扩展 = 从锚点向目标方向逐步扩展

---

## 九、批量操作执行顺序

不同操作需要不同的遍历顺序以保证正确性：

| 操作 | 遍历顺序 | 原因 |
|------|---------|------|
| **删除** | 从下往上 | 避免先删上方节点导致下方节点索引偏移 |
| **缩进** | 从上往下 | 上方节点先缩进成为前一兄弟的子节点，下方节点再缩进时能跟随 |
| **取消缩进** | 从下往上 | 下方节点先取消缩进，避免影响上方节点的父子关系 |
| **复制** | 从下往上 | 副本插入在原节点下方，从下往上保证位置正确 |
| **切换 Checkbox** | 顺序无关 | 各节点独立切换 |

## 十、Zoom 切换与选择

当用户切换 Zoom（聚焦到某个节点的子树视图）时：

- **自动清除所有选择**：防止跨视图的残留选择
- 无选择状态下按 **Esc** 会退出 Zoom（PanelStack.pop），回到上一层视图

## 实现考量

### 多选与 FieldValueOutliner 的关系

- 多选仅在主 OutlinerView 中有效
- FieldValueOutliner 内的节点不参与多选
- 在 FieldValueOutliner 上下文中，Shift+Arrow 应触发 `onNavigateOut` 而非扩展选区

### 可见节点范围计算

Shift+Click / 拖动选择需要计算两个节点之间的可见节点列表。使用 `tree-utils.ts` 中的 `flattenVisibleNodes` 工具函数，结合 `expandedNodes` 状态确定哪些子树是展开的。

### 拖动选择的 document 级监听

contenteditable 元素会捕获鼠标事件，导致父容器上的 React onMouseMove 不可靠触发。拖选必须在 document 级别注册 mousemove/mouseup 监听器。
