# AI-First Product Vision

> 从 "工具 + AI 辅助" 翻转为 "AI + 工具辅助"

## 核心洞察

用户不需要更好的笔记工具，用户需要一个记得住所有东西、想得起该想的、说得出有用的话的第二大脑。节点树是这个大脑的神经元，Chat 是它的嘴。嘴应该是默认界面，神经元应该在幕后。

## 用户的真正问题

用户的核心痛点不是"如何记录"，是"想不起来"：

- 记得之前看过一篇文章，但找不到了
- 知道对某个话题有过思考，但想不起细节
- 收藏了一百篇文章，从来没有回顾过

**记录只是手段，回忆才是目的。**

当前"工具优先"模式要求用户持续投入组织成本（标签、字段、视图），但愿意精心组织信息的用户 < 1%。这不是用户的问题，是产品模型的问题。

## 交互翻转

|  | 工具优先 | AI 优先 |
|---|---------|---------|
| 记录 | 用户手动组织结构 | 用户随便扔进去，AI 自动结构化 |
| 检索 | 用户构造搜索条件 | 用户用自然语言描述需求 |
| 关联 | 用户手动链接 | AI 主动浮现关联 |
| 回顾 | 用户主动打开笔记 | AI 在合适的时机推送相关内容 |
| 学习曲线 | 需要理解标签/字段/视图 | 说话就行 |

**本质区别**：工具优先要求用户适应系统的数据模型；AI 优先让系统适应用户的思维方式。

## 三层体验模型

### 第一层：零成本输入（已有）

浏览网页时一键剪藏、随手写一句话、高亮一段文字。soma 已经做得不错。

### 第二层：不问就给（当前缺失，最大机会）

- 在看一篇关于 pricing 的文章，侧边栏自动浮现之前关于 pricing 的所有笔记
- 创建了一个新节点，AI 自动建议关联的已有节点
- 一周没打开 soma，打开时自动呈现"你可能忘了但很重要的"内容
- **用户不需要做任何操作就获得价值**

### 第三层：问了就答（Chat 当前能力）

- "帮我总结上周所有关于竞品分析的笔记"
- "我之前记过一个定价公式，帮我找出来"
- "把这篇文章的要点和我之前的研究做对比"

## Chat 作为主界面

Chat 作为默认界面，Node Panel 作为可选的深入操作层。

```
┌─────────────────────────────────┐
│  Chat (主界面/默认层)             │
│  对话式交互，AI 主动 + 被动      │
├─────────────────────────────────┤
│  Node Panel (可选的可视化层)      │
│  用户想直接操作时打开             │
│  大纲、字段、标签 — 全部保留      │
└─────────────────────────────────┤
│  LoroDoc (数据层，对用户不可见)    │
└─────────────────────────────────┘
```

## Chat 内嵌节点渲染

AI 的回答不只是文本，而是可以直接嵌入活的节点。采用**渐进式交互**：

### 默认态：紧凑预览

标题 + bullet 列表，和 Chat 文本自然混排。

### 点击：展开详情

在 Chat 内展开 children，本质是嵌入的 mini OutlinerView。

### 双击/按钮：跳到面板

用户觉得"我要认真看这个" → 打开到独立 NodePanel。

**三层是同一个节点的三种呈现密度，用户通过自然交互逐步深入。从 Chat 开始，需要时无缝过渡到面板。**

### 数据原则

Chat 中嵌入的是**真实节点引用**，不是快照副本：
- AI 调用 `node_search` → 返回结果节点 ID 列表 → Chat 渲染这些节点
- AI 调用 `node_create` → 创建真实节点树 → Chat 渲染新创建的节点
- Chat 里看到的和 Panel 里看到的永远是同一份数据，无同步问题

## 待研究：节点编辑与模型上下文同步

**问题**：用户在 Chat 中编辑了 AI 返回的节点（修改内容、合并节点、删除节点），模型的上下文还停留在旧状态。

**推荐方向**：不追踪操作过程，在 `transformContext` 阶段自动注入当前对话引用过的节点的**实时状态**。类似 Cursor 的 file context——编辑器内容是实时注入的，不靠历史消息同步。

**实现思路**：
1. 扫描历史消息，收集所有被引用/创建/搜索到的节点 ID
2. 批量读取最新状态
3. 作为动态上下文注入

详细设计待后续深入。

## 与现有架构的关系

当前架构已经很接近：
- Node 层是独立数据层（LoroDoc + CRDT）
- Agent 已有 6 个 node tools（CRUD + search + undo）
- Chat 已是面板级，和 NodePanel 同权
- #agent / #skill 配置全部是节点
- `transformContext` 管线已支持动态上下文注入

翻转不需要重写，而是**重新排列优先级**：Chat 从配角变主角，Node Panel 从主角变工具箱。
