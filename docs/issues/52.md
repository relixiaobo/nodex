# Issue #52: @ reference 下拉菜单 Enter/点击无法选中项 + ↑/↓ 始终跳到第一个

## 症状

1. 输入 `@im` → 下拉菜单出现 → 按 ↑/↓ → 选中项始终停留在第一个
2. 按 Enter 确认 → 不执行任何操作（节点不变，下拉菜单不关闭）
3. 反复按 Enter 无效

## 已完成的修复（部分有效）

### Fix 1: triggerStateRef 替代 callbacksRef（已合入）
- `NodeEditor.tsx` 中 5 个键盘意图解析点改用 `triggerStateRef.current` 而非 `callbacksRef.current`
- **效果**: Enter 意图解析正确路由到 `reference_confirm`（日志确认）

### Fix 2: prevRefQueryRef 防重复重置（已合入）
- `handleReference` 仅在 query 变化时才 `setRefSelectedIndex(0)`
- **效果**: 理论上应阻止 evaluateTriggers keyUp 回调重置 index，但日志显示 getSelectedItem() 仍返回第一个

## 两个残留子问题

### 子问题 A: selectedIndex 不递增

**现象**: 按 ArrowDown 2 次后 Enter，`getSelectedItem()` 返回 `subtask_2a`（始终第一个），itemCount=5

**可能原因**:
1. `handleReferenceNavDown` 中 `refDropdownRef.current?.getItemCount()` 返回 0（ref 尚未连接）→ 功能更新器返回 0
2. React 19 自动批处理导致 keydown (setIndex) 和 keyup (evaluateTriggers) 在同一批次 → index 被覆盖
3. `useImperativeHandle` 的 deps 变化未触发 ref 更新

**建议诊断**: 在 `handleReferenceNavDown` 中添加 `console.log`:
```typescript
const handleReferenceNavDown = useCallback(() => {
    setRefSelectedIndex((i) => {
      const count = refDropdownRef.current?.getItemCount() ?? 0;
      const next = count > 0 ? Math.min(i + 1, count - 1) : 0;
      console.log('[DBG-refNavDown]', { prev: i, count, next });
      return next;
    });
  }, []);
```

### 子问题 B: handleReferenceSelect 静默失败

**现象**: Enter 确认后 `refConfirm` 日志显示 `item={type:"existing", id:"subtask_2a"}`，但节点未变化，下拉菜单未关闭。用户连续按 8 次 Enter 无效。

**可能原因**:
1. `handleReferenceSelect` 中 `editorRef.current` 为 null → 静默 return
2. `wsId` 或 `userId` 为 falsy → 静默 return
3. `refRangeRef.current` 范围无效 → 文本切片校验失败 → 进入 fallback 但仍失败

**建议诊断**: 在 `handleReferenceSelect` 入口添加 `console.log`:
```typescript
console.log('[DBG-refSelect]', {
  refNodeId, wsId, userId,
  hasEditor: !!editorRef.current,
  refRange: refRangeRef.current,
  fullText: editorRef.current?.getText(),
});
```

## 修复方案（02-14 已合入）

### Sub-problem A Fix: Nav index 不递增

**根因**: `handleReferenceNavDown` 在 React state updater 函数内部读取 `refDropdownRef.current?.getItemCount()`，在 React 19 的 StrictMode 或 state 批处理中，ref 读取可能返回过时值（0），导致 updater 始终返回 0。

**修复**: 将 `getItemCount()` 调用移到 state updater 外部，在同步事件处理上下文中读取 ref：
```typescript
// Before:
setRefSelectedIndex((i) => {
  const count = refDropdownRef.current?.getItemCount() ?? 0;
  return count > 0 ? Math.min(i + 1, count - 1) : 0;
});

// After:
const count = refDropdownRef.current?.getItemCount() ?? 0;
setRefSelectedIndex((i) => count > 0 ? Math.min(i + 1, count - 1) : i);
```

同样修复了 `handleHashTagNavDown`。

### Sub-problem B Fix: handleReferenceSelect 静默失败

**根因**: `handleReferenceSelect` 中的 `if (!ed) return;` 早期退出不关闭下拉菜单，导致用户反复按 Enter 无响应。

**修复**: 将 dropdown 关闭逻辑提取为 `closeDropdown()` 函数，在所有退出路径（包括 early return）中调用。
