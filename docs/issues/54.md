# Issue #54: Schema 空子节点 Backspace 无法删除

## 症状

展开 Schema 节点 → 创建一个空子节点 → 按 Backspace → 节点不被删除。其他父节点的空子节点可以正常删除。

附带问题：Schema 节点在每次进入 workspace 时都是展开状态（这是 localStorage 持久化的 expandedNodes，非代码 bug。清除 `localStorage` 中 `nodex-ui` key 即可重置）。

## 已完成的修复

### Fix: \u200B 零宽空格 isEmpty 判断（已合入）

```typescript
// NodeEditor.tsx Backspace handler ~line 489
// Before:
const isEmpty = !(root?.textContent ?? '').trim().length;
// After:
const isEmpty = !(root?.textContent ?? '').replace(/\u200B/g, '').trim().length;
```

对应 Vitest 回归测试：`tests/vitest/editor-isEmpty.test.ts`（7 cases）

## Backspace 完整代码路径

```
用户在空节点按 Backspace
  → NodeEditor.handleKeyDown
  → caret === 0 && !hasSelection
  → callbacksRef.current.onBackspaceAtStart()  [= handleBackspaceAtStart]
  → 检查 _ownerId === parentId ✓（Schema 是 owner）
  → 检查 !_docType ✓（新建子节点无 docType）
  → index > 0 ✓（如果不是第一个子节点）
  → prevNode._docType = 'tagDef' → return false ← 前兄弟是 tagDef，无法 merge

  (merge 失败, 回到 NodeEditor)
  → isEmpty = !(rawText.replace(/\u200B/g, '')).trim().length
  → isEmpty = true → onDelete() [= handleDelete]
  → textOnly = currentName.replace(/<[^>]*>/g, '').trim()
  → textOnly.length === 0 → trashNode(nodeId, wsId, userId)
```

### 可能的额外失败点

1. **Schema 的 children 中新建节点是第一个位置 (index=0)**: `handleBackspaceAtStart` 中 `index <= 0` 返回 false → 无法 merge（这是正确的）→ 但 isEmpty 路径仍应工作
2. **`htmlToPlainText` 和 `replace(/<[^>]*>/g, '')` 对 \u200B 处理不一致**: `handleDelete` 用 `replace(/<[^>]*>/g, '').trim()` 去 HTML 标签但不去 \u200B → `textOnly = '\u200B'` → `textOnly.length === 1 > 0` → **不删除！**

### 关键发现：handleDelete 也需要 \u200B 处理

```typescript
// OutlinerItem.tsx handleDelete ~line 657
const textOnly = currentName.replace(/<[^>]*>/g, '').trim();
// 如果 name 是 '\u200B'，textOnly = '\u200B'，length = 1 → 不删除
```

**修复方案**:
```typescript
const textOnly = currentName.replace(/<[^>]*>/g, '').replace(/\u200B/g, '').trim();
```

## 修复方案（02-14 已合入）

**根因**: `handleDelete` 在 OutlinerItem.tsx 中用 `replace(/<[^>]*>/g, '').trim()` 去除 HTML 标签，但不去除 `\u200B`（零宽空格）。当节点名是 `'\u200B'` 时，`textOnly = '\u200B'`，长度为 1，`handleDelete` 判断为非空而拒绝删除。

**修复**: 在 `handleDelete` 的 `textOnly` 计算中增加 `.replace(/\u200B/g, '')`:
```typescript
const textOnly = currentName.replace(/<[^>]*>/g, '').replace(/\u200B/g, '').trim();
```

此修复已在上一轮部分修复中合入。本次清理了 `[DBG-handleDelete]` 和 `[DBG-Backspace]` 诊断日志。
