# soma Product Vision v2

> Think with your AI.

## 一、产品形态

一个输入框。用户想到什么就写什么——记录、提问、指令、随想——AI 理解意图，做出最合适的响应。

```
用户：今天会议决定 Q3 涨价 15%

soma：已记录。
     → 你之前有一条笔记"竞品定价对比"提到竞品 Q2 涨了 10%，
       这次涨幅比竞品更激进，要关注一下吗？
```

```
用户：帮我整理上周的会议笔记

soma：找到 4 条会议笔记，按主题整理如下：
     [内嵌节点列表，可展开、可编辑、可跳转]
```

```
用户：如何看待 AI agent 的发展趋势

soma：基于你之前的 3 条相关笔记和当前页面...
```

**用户不需要区分"记笔记"还是"跟 AI 聊天"。** 写进去，得到有价值的回应。AI 的回应自然地包含记录、连接、分析。

背后有一整个知识图谱（节点树）作为 AI 的记忆。用户偶尔需要时，可以打开 Outliner 直接查看和编辑节点——但这不是默认动作，而是深入操作时的选择。

## 二、核心理念

### 从 v1 到 v2 的演进

v1 围绕 **Think → Connect → Emerge** 三步循环（见 `product-philosophy.md`）。每一步依赖用户主动操作：手动记录 → 手动组织 → 等待涌现。

v2 的核心转变：

| v1 | v2 |
|----|-----|
| Think where you read | Think with your AI |
| 工具 + AI 辅助 | 人机协作 |
| 用户是数据主人，AI 是工具 | 用户和 AI 都是数据主人 |
| Think → Connect → Emerge 串行循环 | 对话中三者一体 |
| 多个功能入口 | 一个输入框 |
| Outliner 是主界面 | 输入框是主界面，Outliner 是可选的深入操作层 |

### 不变的理念

- **工具应该让人变强，而不是让人觉得工具很强** — AI 协作的目标是用户获得新理解
- **知识组织没有客观正确答案** — AI 的"错误"组织可能是惊喜
- **正反馈是成功的关键** — 对话本身就是即时反馈

## 三、人机协作

### AI 是协作伙伴，不是工具

用户和 AI 在同一块画板（节点树）上共同工作。任何一方都可以创建、编辑、连接节点，不需要另一方许可。

- **用户输入** → AI 理解意图，执行最合适的动作（记录 / 搜索 / 分析 / 操作）
- **AI 主动行动** → 发现重要内容时创建节点、发现连接时建立关联
- **一切可审查、可修正、可删除** — 协作不等于失控

### 区分来源，不区分权限

AI 创建的内容标记来源（如小标识），用户可以确认、编辑、删除。编辑过的变成共创内容。节点树从"用户的笔记本"变成**人机共同的思考痕迹**。

### 对话即思考

不再是先记录再组织再等待涌现——对话中三者同时发生：

- 用户带着想法来对话 → **记录**（Think）
- AI 拉入相关笔记，发现连接 → **组织**（Connect）
- 用户看到意想不到的关联 → **涌现**（Emerge）
- 有价值的部分自动沉淀为节点 → 下次对话时被 AI 拉入

**飞轮：对话 → 沉淀 → 对话 → 沉淀。** 每次对话都在丰富共享画板，画板又让下次对话更深入。

## 四、认知摩擦

### 为什么需要摩擦

AI 协作中最省力的路径是"让 AI 做完，我看结果"。但这让用户变成消费者——没有思考，没有成长。

soma 的独特定位：**在正确的地方主动增加认知摩擦。** 不是阻碍用户，而是邀请用户做一个判断。判断 = 思考 = 成长。

### 摩擦的触发原则

> **摩擦带来的收获 > 用户被打断的成本，才触发。**

不是有机会就触发。大部分时候 AI 应该安静地执行——记录、关联、沉淀。只有当 AI 发现了**用户大概率会觉得有价值**的连接或矛盾时，才邀请判断。

好的摩擦：
- "这跟你上周记的 X 好像矛盾？" → 用户发现了自己观点的变化（高收获）
- "你最近三次讨论都提到了 Y，值得提炼成一个主题吗？" → 用户看见了自己的模式（高收获）

坏的摩擦：
- 每次记录都问"要不要加标签？" → 打断流畅的记录（高成本，低收获）
- 每次搜索都问"哪些最相关？" → 用户已经在看了，不需要被问（无意义）

**形式是选择题，不是问答题。** 不是让用户从零构建，而是在 AI 给出的结果中做判断。

### 摩擦的度

| 太少 | 恰到好处 | 太多 |
|------|---------|------|
| AI 做完一切，用户只看 | AI 做结构化部分，在高价值时刻邀请判断 | 每次操作都要确认 |
| 用户没有成长 | 用户在判断中获得新理解 | 用户烦了就不用了 |
| ≈ 纯 ChatGPT | ≈ soma | ≈ Clippy |

## 五、产品架构

### 对话界面 + 静默输入 + 共享画板

```
输入渠道:
  1. Chat 对话 (主入口，侧边栏打开时)
     写什么都行，AI 理解意图
  2. 网页高亮/笔记/剪藏 (静默输入，侧边栏无需打开)
     在网页上直接操作，零摩擦，无需 AI 参与

共享画板:
  LoroDoc 节点树 — 两个渠道的内容都沉淀在这里

输出界面:
  Chat — 所有价值通过对话兑现
  Outliner — 可选的深入操作
```

### 两种输入渠道

**Chat 对话**（主入口）：用户打开侧边栏，看到对话界面。写什么都行——记录、提问、指令、随想。AI 理解意图，做出最合适的回应。

**静默输入**（辅助入口）：高亮、笔记、剪藏是在网页上的空间操作，通过 Content Script 完成，**侧边栏无需打开**。没有 AI 参与，没有对话回应——纯粹的零摩擦输入。

静默输入的价值不在操作时兑现，而在**之后的对话中兑现**：
- 用户高亮了一段关于定价的文字（静默，无打扰）
- 三天后用户在 Chat 中讨论定价策略 → AI 自然拉入这条高亮
- 用户说"最近我看了什么关于 X 的" → AI 能答出来

**静默输入丰富 AI 的记忆，对话是记忆兑现价值的出口。**

### Chat 输入框增强：@ 引用节点

输入框支持 `@` 触发节点选择器，让用户精确地围绕某些节点聊天：

```
用户：帮我对比 @竞品定价对比 和 @Q3涨价决策 的矛盾点
用户：基于 @AI Agent 架构笔记 扩展关于多 Agent 编排的思路
用户：@今天的会议记录 提取 action items
```

输入 `@` → 弹出节点搜索下拉 → 选中后插入引用 → 节点内容作为精确上下文注入。类似 GitHub `@mention` 或 Slack `@user`，在原生 textarea 上实现（浮动下拉 + 引用标记），不需要富文本编辑器。

### 回应中的内嵌节点

AI 的回答不只是文本，而是可以嵌入活的节点。渐进式交互：

1. **默认态** — 标题 + bullet，和文本自然混排
2. **点击** — 展开 children（mini OutlinerView）
3. **按钮** — 跳到独立 Outliner 面板

嵌入的是真实节点引用，不是快照。Chat 和 Outliner 看到的永远是同一份数据。

### 浏览器上下文

soma 活在浏览器侧边栏——天然知道用户正在看什么。这个上下文自动注入对话，不需要用户解释。

- 用户在看一篇文章，直接问"这篇文章的要点" → AI 已经知道是哪篇
- 用户在看一个产品页，说"记一下这个定价方案" → AI 知道具体内容
- 用户在看自己之前的笔记，问"帮我扩展一下这个想法" → AI 知道上下文

### 预设召唤

用户主动设置持续运行的指令：
- "每天早上总结昨天的笔记"
- "浏览 AI 相关文章时提醒我相关笔记"
- "每周回顾 #research 下的新增内容"

不是 AI 自作主张推送，而是用户授权的自动化。

## 六、与 v1 产品哲学的关系

### 保留

- 核心价值观（让人变强、正反馈、少即是多）
- 浏览器侧边栏的位置优势
- "一切皆节点" 的数据模型
- 高亮/剪藏的零成本输入
- Supertag + Field 的结构化能力
- Spark 认知触发

### 演进

- **Think → Connect → Emerge** 不再是显式的三步循环，而是在对话中自然融合
- **"AI 只在用户召唤时行动"** 演进为 AI 可在协作中主动行动（标记来源，用户可审查）
- **多功能入口** 收敛为一个输入框 + 浏览器空间操作（高亮/剪藏）
- **Outliner 作为主界面** 演进为 Outliner 作为可选的深入操作层

### 不变的设计红线

1. **用户永远可以覆盖 AI 的任何决定**
2. **AI 创建的内容标记来源**
3. **每次交互包含至少一个用户判断点**
4. **笔记不够多时坦诚说"再积累一些"**
5. **预设召唤需要用户主动设置**

## 七、产品理念演进历程

### v1：记下来，用得上

> Slogan: "记下来，用得上"

- 方法论：**捕获 → 消化 → 重遇**（三步线性流）
- 核心假设：笔记是对信息的预处理
- AI 角色：未明确
- 主界面：Outliner

### v2：Think where you read

> Slogan: "Think where you read"

- 方法论：**Think → Connect → Compound**
- 关键变化：从"记录"升级为"思考"，"重遇"变成"复利（Compound）"
- 新增：费曼学习法理念——"用自己的话写才算理解"
- AI 角色：辅助

### v3：AI 策略对齐

> Slogan: "Think where you read"

- 方法论：**Think → Connect → Compound**（不变）
- 关键变化："选择是思考的第一步"（高亮即思考）、三条认知工作流（阅读环/碰撞环/回顾环）
- 新增：开放问题区（剪藏边界、上下文感知、AI 工作流决策）

### v4：AI 照亮你的思考

> Slogan: "Think where you read"

- 方法论：**Think → Connect → Emerge**（Compound → Emerge，强调涌现）
- 关键变化：AI 独立成章（"照亮你的思考"）、干/湿分工、Outliner = 共享画板、Taste 学习、设计红线
- 新增：市场定位图、产品路径（4 条）、"知识组织没有客观正确答案"
- 文档：`product-philosophy.md`（当前版本）

### v5：人机协作（当前方向）

> Slogan: "Think with your AI"

- 方法论：**对话即思考**（Think/Connect/Emerge 在对话中一体化）
- 关键变化：
  - 一个输入框替代多入口
  - AI 从工具变为协作伙伴（也是数据主人）
  - 认知摩擦主动设计
  - Outliner 从主界面变为可选深入操作层
  - 记录和对话的边界消失
- 文档：`ai-first-product-vision.md`（本文件）

### 演进脉络

```
v1  记录工具    "记下来，用得上"           捕获 → 消化 → 重遇
v2  思考工具    "Think where you read"    Think → Connect → Compound
v3  +AI 策略    "Think where you read"    + 认知工作流
v4  AI 照亮     "Think where you read"    Think → Connect → Emerge
v5  人机协作    "Think with your AI"      对话即思考
```

从"记录"到"思考"到"协作"——每次迭代都在把更多的认知工作从用户独自承担，转移到人机共同完成。核心不变的是：**工具让人变强，不是让人觉得工具强。**
