# soma 产品方向：AI-First

> Think with your AI.

## 一、产品形态

用户选择自己的体验模式——Chat-first（与 AI 思考）或 Outliner-first（结构化笔记）。AI 是可选的增强，Outliner 独立可用。

新用户首次打开 soma，登录后进入 Chat。填写 API key 开始与 AI 对话，或切换到 Outliner 模式直接记笔记。选择保存在 Settings 中，下次自动进入偏好的模式。

两种模式不互斥——Chat 用户随时可以打开 Outliner 查看和编辑节点，Outliner 用户随时可以打开 Chat 与 AI 对话。

**用户不需要区分"记笔记"还是"跟 AI 聊天"。** 写进去，得到有价值的回应。

```
用户：今天会议决定 Q3 涨价 15%

soma：已记录。
     → 你之前有一条笔记"竞品定价对比"提到竞品 Q2 涨了 10%，
       这次涨幅比竞品更激进。你怎么看这个差距？
```

```
用户：帮我整理上周的会议笔记

soma：找到 4 条会议笔记，按主题整理如下：
     [内嵌节点列表，可展开、可编辑、可跳转]
```

```
用户：如何看待 AI agent 的发展趋势

soma：基于你之前的 3 条相关笔记和当前页面...
     不过有个角度你可能没考虑到——
```

## 二、核心理念

### AI 的身份：批判性思考伙伴

soma 的 AI 不是助手，不是工具，是**思考伙伴**。伙伴会质疑你的假设、提供不同角度、指出你忽略的矛盾——不是为了反对你，而是为了帮你看得更清楚。

> 你的工作是帮用户看清楚，即使这意味着质疑他们的假设。

AI 的身份和能力遵循 **is/has 框架**——跟节点系统的 supertag/field 一样的设计哲学：

| 层 | 类比 | 内容 | 可修改性 |
|---|---|---|---|
| **System prompt（is）** | supertag | 身份、价值观、沟通格式 | 代码管理 |
| **Skill（has）** | field | 具体能力、行为模式 | 用户可调整 |

**System prompt** 定义 AI 是谁——批判性思考伙伴、诚实、跟用户一起成长。去掉这些，AI 就不是它自己了。

**Skill** 定义 AI 能做什么——管理节点、搜索历史对话、帮用户创建新技能。去掉某个 skill，AI 少了一个能力，但还是它自己。

### AI 跟用户一起成长

AI 的配置是知识图谱中的一个节点（`#agent`）。它的子节点是持久化指令——用户说"以后用中文回复"，AI 就把这条写进自己的配置节点，下次新对话自动生效。

用户不需要知道 `#agent` 节点的存在。他们只需要跟 AI 说"改变你的行为"，AI 自己完成。

### 三个内置 skill

| Skill | 做什么 | 默认激活 |
|-------|--------|---------|
| **Node organizer** | 记录、标签、沉淀、发现关联 | ✅ |
| **Chat recall** | 搜索和回忆历史对话 | ✅ |
| **Skill creator** | 帮用户创建/编辑新的 #skill 节点 | ✅ |

用户可以在 Settings > AI > Skills 中看到、编辑、关闭这些 skill，也可以创建自己的。

### 不变的信念

- **工具应该让人变强，而不是让人觉得工具很强** — AI 协作的目标是用户获得新理解
- **知识组织没有客观正确答案** — AI 的"错误"组织可能是惊喜
- **正反馈是成功的关键** — 对话本身就是即时反馈
- **诚实** — 不知道就说不知道，数据不够就说不够，不编造关联

### 保留的理念

- 浏览器侧边栏的位置优势
- "一切皆节点" 的数据模型
- 高亮 / 剪藏的零成本输入
- Supertag + Field 的结构化能力
- Spark 认知触发

## 三、人机协作

### 对话即思考

不再是先记录再组织再等待涌现，对话中三者同时发生：

- 用户带着想法来对话 → **记录**（Think）
- AI 拉入相关笔记，发现连接 → **组织**（Connect）
- 用户看到意想不到的关联 → **涌现**（Emerge）
- 有价值的部分自动沉淀为节点 → 下次对话时被 AI 拉入

**飞轮：对话 → 沉淀 → 整理 → 更深的发现 → 下次对话更深入。**

### 共享画板

用户和 AI 在同一块画板（节点树）上共同工作。任何一方都可以创建、编辑、连接节点。

- **用户输入** → AI 理解意图，执行最合适的动作
- **AI 主动行动** → 发现重要内容时创建节点、发现连接时建立关联
- **一切可审查、可修正、可删除** — 协作不等于失控

AI 创建的内容标记来源，用户可以确认、编辑、删除。节点树从"用户的笔记本"变成**人机共同的思考痕迹**。

### 沉淀规则

AI 沉淀的内容跟用户手动记录的没有结构上的区别——遵守同样的节点系统规则：

1. **先找已有标签**：复用用户建立的语义体系（标签 = is，字段 = has）
2. **没有合适的就新建标签**：让标签体系随使用自然生长
3. **无标签也行**：不是所有想法都需要分类
4. **字段同理**：选择跟标签匹配的已有字段

**保守原则**：只沉淀决策、结论、长期偏好。日常对话细节不沉淀。宁可少记，不要膨胀 outliner。

### 沉淀生命周期

```
对话中产生 → 沉淀为节点 → 定期整理 → 提炼/合并/淘汰
```

AI 定期审视近期沉淀的节点，发现模式、合并重复、淘汰过时的，把有价值的提炼为更持久的节点。

## 四、认知摩擦

### 为什么需要摩擦

AI 协作中最省力的路径是"让 AI 做完，我看结果"。但这让用户变成消费者，没有思考，也没有成长。

soma 的独特定位：**在正确的地方主动增加认知摩擦。** 不是阻碍用户，而是邀请用户做一个判断。判断 = 思考 = 成长。

**摩擦不是为了确认操作，而是为了放大思考。**

### 摩擦的触发原则

> **摩擦带来的收获 > 用户被打断的成本，才触发。**

大部分时候 AI 应该安静地执行。只有当 AI 发现了**值得用户判断的认知信号**时，才递出一个思考机会：

- 这和你之前的观点可能矛盾
- 你最近反复回到同一个主题
- 两个原本分离的主题可能有交集
- 当前结论背后缺一个关键证据
- 你已经积累到足以提炼出更高层概念

**形式是思考入口，不是审批流程。** 轻量提示、选择题、建议分支——不是 modal、确认框。

### 浏览文章 — 静默输入 + 对话

**不打开侧边栏**：选中一段文字 → 高亮工具栏 → 点高亮 → 存节点 → 继续阅读。完全静默。

**打开侧边栏**：

```
用户：这篇文章的核心观点是什么

soma：这篇文章对比了 5 个 AI Agent 平台...

     跟你之前的 @AI Agent 服务商调研 有交集——
     那条笔记里你已经分析了 Okara 和 CrewAI，
     这篇新增了 AutoGen 和 LangGraph 的对比。
     要合并到一起吗？
```

### 快速记录 — AI 安静地记

```
用户：定价策略的核心不是价格本身，而是用户对价值的感知

soma：已记录。
     → 这跟你上周高亮的一段话呼应——
       "Stripe 的定价页不展示价格，展示的是每个方案能帮你做什么"
```

大部分时候安静确认。只有发现高价值关联时才补充。没有关联时不强行凑。

### 回顾 — 对话中自然涌现

```
用户：这周我都在想什么

soma：这周你的笔记集中在两个主题：

     1. 定价策略（8 条）— 从竞品调研到涨价决策
     2. AI Agent 生态（5 条）— 从服务商调研到架构思考

     有意思的是，这两个主题可能有交集——
     你记了一条"Agent 服务商的定价模式值得研究"，
     但还没有展开。要深入聊聊吗？
```

## 六、产品架构

### 总览

```
输入渠道:
  1. Chat 对话（默认入口，可切换）
     记录、提问、指令、随想都从这里进入
  2. Outliner（可选入口）
     结构化笔记、树操作、标签管理
  3. 网页高亮 / 笔记 / 剪藏（静默输入，侧边栏无需打开）
     在网页上直接操作，零摩擦，无需 AI 参与

共享画板:
  LoroDoc 节点树 — 所有渠道的内容都沉淀在这里

输出界面:
  Chat — 对话中兑现价值
  Outliner — 结构化查看与编辑
  两者看到的是同一份数据
```

### 静默输入

高亮、笔记、剪藏是在网页上的空间操作，**侧边栏无需打开**。

静默输入的价值不在操作时兑现，而在**之后的对话中兑现**：
- 用户高亮了一段关于定价的文字（静默，无打扰）
- 三天后用户在 Chat 中讨论定价策略 → AI 自然拉入这条高亮

**静默输入丰富 AI 的记忆，对话是记忆兑现价值的出口。**

### 记忆分层

soma 不应出现一个用户不可见的独立 memory 层。记忆分三层：

- **Chat transcript** — 协作过程层。来回讨论、判断过程、未完成的脉络。独立于节点图谱，但不是知识的最终落点
- **Outliner / 节点树** — 长期记忆层。凡是用户应该能看到、修正、引用的记忆，都是节点
- **Runtime retrieval / cache** — 运行时辅助层。检索索引、摘要缓存。不是用户意义上的"记忆"

判断标准：**凡是用户可感知、可审查、可修正的长期记忆，都必须落为节点。**

### 工具层

8 个 AI 工具：node_create / node_read / node_edit / node_delete / node_search / undo / browser / past_chats。

- **node_read** 支持无参数浏览——从工作区根节点开始探索知识图谱，支持 `"journal"` / `"schema"` 快捷名称
- **past_chats** 采用三层渐进探索：session 列表 → user 消息列表 → 消息详情（分页）
- **node_search** 使用 uFuzzy 模糊搜索，支持中文多词查询

### Chat 中的节点展示

三种节点呈现方式：

| markup | 用途 | 渲染 |
|--------|------|------|
| `<ref id="xxx">文本</ref>` | 行内提及 | 蓝色链接 → 点击弹 Popover |
| `<cite type="TYPE" id="ID">N</cite>` | 溯源引注 | 灰色角标 → 点击弹 Popover |
| `<node id="xxx" />` | 展示节点内容 | 内嵌 OutlinerView（可展开、可编辑） |

cite 支持三种来源类型：`type="node"`（知识图谱节点）、`type="chat"`（历史对话）、`type="url"`（网页）。

### @ 引用节点

输入框支持 `@` 触发节点选择器，让用户精确地围绕某些节点聊天。

### 浏览器上下文

soma 活在浏览器侧边栏，天然知道用户正在看什么。这个上下文自动注入对话。

### 预设召唤

用户主动设置持续运行的指令——用户授权的自动化，不是 AI 自作主张。

## 七、异步任务与多 Agent 协作

### 任务即节点

`#task` supertag + `status` 字段（pending / running / done / failed）。任何节点打上 `#task` 就自动拥有状态。

### 异步任务的运行模式

当前架构：AI 请求从 Side Panel 发出，关闭则终止。异步任务 = 侧边栏保持开着 + AI 自主跑多轮 tool call + tab group 隔离浏览器操作。

### 多 Agent 协作

异步能力是前置条件。多个 Agent 并行执行子任务，编排层协调分配和依赖。

## 八、设计红线

1. **用户永远可以覆盖 AI 的任何决定**
2. **AI 创建的内容标记来源**
3. **高价值时刻创造判断机会，日常操作不打扰**
4. **数据不够时坦诚说"再积累一些"**
5. **预设召唤需要用户主动设置**
6. **用户可感知的长期记忆必须是节点**
7. **AI 沉淀遵守跟用户一样的节点系统规则**
8. **AI 是批判性思考伙伴，不是应声虫**
9. **AI 的能力是模块化的（skill），用户可以调整**
10. **AI 跟用户一起成长，通过 #agent 节点自我进化**

## 十、演进历程

```
v1  记录工具    "记下来，用得上"           捕获 → 消化 → 重遇
v2  思考工具    "Think where you read"    Think → Connect → Compound
v3  +AI 策略    "Think where you read"    + 认知工作流
v4  AI 照亮     "Think where you read"    Think → Connect → Emerge
v5  人机协作    "Think with your AI"      对话即思考
v6  思考伙伴    "Think with your AI"      批判性思考 + 模块化能力 + 自我进化
```

从"记录"到"思考"到"协作"到"伙伴"——每次迭代都在把更多的认知工作从用户独自承担，转移到人机共同完成。核心不变的是：**工具让人变强，不是让人觉得工具强。**
