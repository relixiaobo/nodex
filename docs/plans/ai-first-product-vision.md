# soma 产品方向：AI-First

> Think with your AI.

## 一、产品形态

Chat 是主界面，Outliner 是可选的查看/编辑层。

用户通过对话思考，不通过组织思考。写什么都行——记录、提问、指令、随想——AI 理解意图，做出最合适的响应。背后有一整个知识图谱（节点树）作为 AI 的记忆。用户偶尔需要时，可以打开 Outliner 直接查看和编辑节点，但这不是默认动作，而是深入操作时的选择。

| 层 | 角色 | 投入优先级 |
|---|---|---|
| **Chat** | 思考发生的地方，主交互面 | 最高 |
| **Node tools** | AI 操作知识图谱的手脚 | 最高 |
| **Outliner** | 查看产物、偶尔手动编辑 | 维护现状，不再大量投入 |

**用户不需要区分"记笔记"还是"跟 AI 聊天"。** 写进去，得到有价值的回应。AI 的回应自然地包含记录、连接、分析。

```
用户：今天会议决定 Q3 涨价 15%

soma：已记录。
     → 你之前有一条笔记"竞品定价对比"提到竞品 Q2 涨了 10%，
       这次涨幅比竞品更激进，要关注一下吗？
```

```
用户：帮我整理上周的会议笔记

soma：找到 4 条会议笔记，按主题整理如下：
     [内嵌节点列表，可展开、可编辑、可跳转]
```

```
用户：如何看待 AI agent 的发展趋势

soma：基于你之前的 3 条相关笔记和当前页面...
```

## 二、核心理念

### 从 v4 到当前方向的演进

v4 围绕 **Think → Connect → Emerge** 三步循环（见 `product-philosophy.md`）。每一步依赖用户主动操作：手动记录 → 手动组织 → 等待涌现。

当前方向的核心转变：

| v4 (之前) | 当前方向 |
|-----------|---------|
| Think where you read | Think with your AI |
| 工具 + AI 辅助 | 人机协作 |
| Think → Connect → Emerge 串行循环 | 对话中三者一体 |
| 多个功能入口 | 一个意图入口 |
| Outliner 是主界面 | Chat 是主界面，Outliner 是可选的深入操作层 |

### 保留的理念

- 核心价值观（让人变强、正反馈、少即是多）
- 浏览器侧边栏的位置优势
- "一切皆节点" 的数据模型
- 高亮 / 剪藏的零成本输入
- Supertag + Field 的结构化能力
- Spark 认知触发

### 不变的信念

- **工具应该让人变强，而不是让人觉得工具很强** — AI 协作的目标是用户获得新理解
- **知识组织没有客观正确答案** — AI 的"错误"组织可能是惊喜
- **正反馈是成功的关键** — 对话本身就是即时反馈

## 三、人机协作

### AI 是协作伙伴，不是工具

用户和 AI 在同一块画板（节点树）上共同工作。任何一方都可以创建、编辑、连接节点。

- **用户输入** → AI 理解意图，执行最合适的动作（记录 / 搜索 / 分析 / 操作）
- **AI 主动行动** → 发现重要内容时创建节点、发现连接时建立关联
- **一切可审查、可修正、可删除** — 协作不等于失控

### 区分来源，不区分行动权

AI 创建的内容标记来源（如小标识），用户可以确认、编辑、删除。编辑过的变成共创内容。节点树从"用户的笔记本"变成**人机共同的思考痕迹**。

### 对话即思考

不再是先记录再组织再等待涌现，对话中三者同时发生：

- 用户带着想法来对话 → **记录**（Think）
- AI 拉入相关笔记，发现连接 → **组织**（Connect）
- 用户看到意想不到的关联 → **涌现**（Emerge）
- 有价值的部分自动沉淀为节点 → 下次对话时被 AI 拉入

**飞轮：对话 → 沉淀 → 对话 → 沉淀。** 每次对话都在丰富共享画板，画板又让下次对话更深入。

### 沉淀规则

AI 沉淀的内容跟用户手动记录的没有结构上的区别——遵守同样的节点系统规则：

1. **先找已有标签**：复用用户建立的语义体系（标签 = is，字段 = has）
2. **没有合适的就新建标签**：让标签体系随使用自然生长
3. **无标签也行**：不是所有想法都需要分类，直接记录即可
4. **字段同理**：选择跟标签匹配的已有字段，而非发明特殊字段

不需要专门的沉淀标签或特殊基础设施。AI 就是一个跟用户一样使用节点系统的协作者。

**保守原则**：只沉淀决策、结论、长期偏好这类跨会话仍然成立的内容。日常对话细节不沉淀。宁可少记，不要膨胀 outliner。

### 沉淀生命周期

沉淀的内容不是"记了就完"，需要定期整理，防止膨胀并提炼更深的发现：

```
对话中产生 → 沉淀为节点 → 定期整理 → 提炼/合并/淘汰
```

AI 定期审视近期沉淀的节点，发现模式、合并重复、淘汰过时的，把有价值的提炼为更持久的节点或更新已有节点。

这是飞轮的完整形态：**对话 → 沉淀 → 整理 → 更深的发现 → 下次对话更深入**。

## 四、认知摩擦

### 为什么需要摩擦

AI 协作中最省力的路径是"让 AI 做完，我看结果"。但这让用户变成消费者，没有思考，也没有成长。

soma 的独特定位：**在正确的地方主动增加认知摩擦。** 不是阻碍用户，而是邀请用户做一个判断。判断 = 思考 = 成长。

**摩擦不是为了确认操作，而是为了放大思考。**

### 摩擦的触发原则

> **摩擦带来的收获 > 用户被打断的成本，才触发。**

不是有机会就触发。大部分时候 AI 应该安静地执行，记录、整理、关联、沉淀都可以直接推进。只有当 AI 发现了**值得用户判断的认知信号**时，才递出一个思考机会。

值得触发摩擦的信号包括：
- 这和你之前的观点可能矛盾
- 你最近反复回到同一个主题
- 两个原本分离的主题可能有交集
- 当前结论背后缺一个关键证据
- 你已经积累到足以提炼出更高层概念

好的摩擦：
- "这跟你上周记的 X 好像矛盾？" → 用户发现了自己观点的变化（高收获）
- "你最近三次讨论都提到了 Y，值得提炼成一个主题吗？" → 用户看见了自己的模式（高收获）

坏的摩擦：
- 每次记录都问"要不要加标签？" → 打断流畅的记录（高成本，低收获）
- 每次搜索都问"哪些最相关？" → 用户已经在看了，不需要被问（无意义）

**形式是思考入口，不是审批流程。** 它通常应该是一个轻量提示、一个选择题、一个建议分支，而不是 modal、确认框或连续 yes/no 问题。主路径始终顺滑，摩擦只在值得的时候点亮。

### 摩擦的度

| 太少 | 恰到好处 | 太多 |
|------|---------|------|
| AI 做完一切，用户只看 | AI 先推进，在高价值时刻邀请判断 | 每次操作都要确认 |
| 用户没有成长 | 用户在判断中获得新理解 | 用户烦了就不用了 |
| ≈ 纯 ChatGPT | ≈ soma | ≈ Clippy |

## 五、使用场景

### 早上 — 打开 soma

用户点击 soma 图标，侧边栏打开。首屏**不依赖 AI 调用**，本地数据即时渲染：

```
┌──────────────────────────────┐
│  soma                    [⚙] │
│                              │
│  Recent                      │
│  • Q3 涨价决策          昨天  │
│  • 竞品定价对比         昨天  │
│  • AI Agent 调研        昨天  │
│                              │
│  ───────────────────────     │
│  Last chat                   │
│  "帮我整理定价相关笔记"       │
│  ───────────────────────     │
│                              │
│  ┌──────────────────────┐    │
│  │ 写点什么...       [@] │    │
│  └──────────────────────┘    │
└──────────────────────────────┘
```

最近的节点 + 最近的对话 + 输入框。用户想浏览就浏览，想聊就聊。打开即可用，无需等待。

### 浏览文章 — 静默输入 + 对话

**不打开侧边栏**：选中一段文字 → 高亮工具栏 → 点高亮 → 存节点 → 继续阅读。完全静默。

**打开侧边栏**：

```
用户：这篇文章的核心观点是什么

soma：这篇文章对比了 5 个 AI Agent 平台...

     跟你之前的 @AI Agent 服务商调研 有交集——
     那条笔记里你已经分析了 Okara 和 CrewAI，
     这篇新增了 AutoGen 和 LangGraph 的对比。
     要合并到一起吗？

用户：好的，合并

soma：已合并到 "AI Agent 服务商调研"：
     ┌─ AI Agent 服务商调研 ────────┐
     │  • Okara.ai — SEO Agent     │
     │  • CrewAI — 多 Agent 框架    │
     │  • AutoGen — 微软开源        │  ← 新增
     │  • LangGraph — LangChain    │  ← 新增
     │  ...                 [打开 ↗] │
     └────────────────────────────┘
```

用户点 [打开 ↗] → Outliner 面板打开，可直接编辑。

### 快速记录 — AI 安静地记

```
用户：定价策略的核心不是价格本身，而是用户对价值的感知

soma：已记录。
```

大部分时候 AI 安静确认。只有发现高价值关联时才补充：

```
soma：已记录。
     → 这跟你上周高亮的一段话呼应——
       "Stripe 的定价页不展示价格，展示的是每个方案能帮你做什么"
```

### 准备工作 — AI 整理 + 内嵌节点

```
用户：帮我整理所有关于 Q3 定价的笔记，下午要汇报

soma：找到 6 条相关笔记，整理为：

     ┌─ Q3 定价汇报准备 ──────────┐
     │  决策                       │
     │  • 涨价 15%（3/18 决定）    │
     │  • 生效时间：Q3 第一天      │
     │                             │
     │  依据                       │
     │  • 竞品 Q2 涨了 10%        │
     │  • 用户调研：价格敏感度低    │
     │                             │
     │  风险                       │
     │  • 企业客户可能比价         │
     │  ...              [打开 ↗]  │
     └────────────────────────────┘

     按"决策/依据/风险"做了分组。
     还需要加什么维度？
```

### 回顾 — 对话中自然涌现

```
用户：这周我都在想什么

soma：这周你的笔记集中在两个主题：

     1. 定价策略（8 条）— 从竞品调研到涨价决策，
        形成了完整的决策链

     2. AI Agent 生态（5 条）— 从服务商调研到
        架构思考，但还没有形成结论

     有意思的是，这两个主题可能有交集——
     你记了一条"Agent 服务商的定价模式值得研究"，
     但还没有展开。要深入聊聊吗？
```

Emerge 不再需要用户主动 `/review`，在对话中自然发生。

## 六、产品架构

### 总览

```
输入渠道:
  1. Chat 对话（主入口，侧边栏打开时）
     记录、提问、指令、随想都从这里进入
  2. 网页高亮 / 笔记 / 剪藏（静默输入，侧边栏无需打开）
     在网页上直接操作，零摩擦，无需 AI 参与

共享画板:
  LoroDoc 节点树 — 两个渠道的内容都沉淀在这里

输出界面:
  Chat — 所有价值主要在对话中兑现
  Outliner — 可选的深入操作与审查层
```

### 静默输入

高亮、笔记、剪藏是在网页上的空间操作，通过 Content Script 完成，**侧边栏无需打开**。没有 AI 参与，没有对话回应，纯粹的零摩擦输入。

静默输入的价值不在操作时兑现，而在**之后的对话中兑现**：
- 用户高亮了一段关于定价的文字（静默，无打扰）
- 三天后用户在 Chat 中讨论定价策略 → AI 自然拉入这条高亮
- 用户说"最近我看了什么关于 X 的" → AI 能答出来

**静默输入丰富 AI 的记忆，对话是记忆兑现价值的出口。**

### 记忆分层

soma 不应出现一个用户不可见的独立 memory 层。记忆分三层：

- **Chat transcript** — 协作过程层。保存"我们怎么走到这里"：来回讨论、判断过程、未完成的执行脉络。独立于节点图谱长期存在，但不是知识的最终落点
- **Outliner / 节点树** — 长期记忆层。凡是用户应该能看到、修正、继承、引用的记忆，都应该是节点：长期偏好、持续主题、未完问题、工作方式约束、任务状态、AI 学到的规则
- **Runtime retrieval / cache** — 运行时辅助层。检索索引、相关性排序、摘要缓存可以独立存在，但不是用户意义上的"记忆"，也不能成为真相源

判断标准：**凡是用户可感知、可审查、可修正的长期记忆，都必须落为节点。**

跨会话能力不是"让模型自动带着所有旧聊天"，而是两步：
1. **检索过去的过程历史** — 按时间找最近聊过什么，或按主题找过去怎么想过
2. **把真正稳定的东西沉淀为节点** — 当某个偏好、约束、主题跨越多次对话仍然成立时，进入 Outliner

### 工具层现状

8 个 AI 工具：node_create / node_read / node_edit / node_delete / node_search / undo / browser / **past_chats**。节点 CRUD 和跨会话记忆均已具备。

past_chats 工具（#157 已完成）采用三层渐进探索设计：session 列表 → user 消息列表 → 消息详情（assistant 回复分页）。AI 自己做相关性判断，关键词过滤是可选加速。详见 `docs/plans/past-chats-tool.md`。

剩余缺口：

| 缺口 | 说明 | 解法 |
|------|------|------|
| 上下文感知不足 | AI 不知道最近访问/编辑了哪些节点 | 丰富 system reminder 注入 |
| 主动沉淀行为 | AI 不知道什么时候该主动记 | system prompt 行为指导（简短即可） |

### @ 引用节点

输入框支持 `@` 触发节点选择器，让用户精确地围绕某些节点聊天：

```
用户：帮我对比 @竞品定价对比 和 @Q3涨价决策 的矛盾点
用户：基于 @AI Agent 架构笔记 扩展关于多 Agent 编排的思路
用户：@今天的会议记录 提取 action items
```

输入 `@` → 弹出节点搜索下拉 → 选中后插入引用 → 节点内容作为精确上下文注入。

### Chat 中的节点展示

Chat 中有三种节点呈现方式，各有不同用途：

| markup | 用途 | 渲染 |
|--------|------|------|
| `<ref id="xxx">文本</ref>` | 行内提及（"你之前有一条笔记..."） | 蓝色链接 → 点击弹 Popover 浮窗 |
| `<cite id="xxx">N</cite>` | 引用证据 | 上标数字 → 点击弹 Popover 浮窗 |
| `<node id="xxx" />` | 展示节点内容 | 内嵌 OutlinerView（可展开、可编辑） |

**浮窗（ref / cite）**：点击弹出 Popover，内容是该节点的 OutlinerView。用户可以在浮窗内展开子节点、编辑内容。底部有"在面板中打开"按钮。不打断对话心流。

**内嵌 outliner（`<node />`）**：AI 回答的一部分，独占一行。复用现有 `OutlinerView` / `OutlinerItem` 组件，样式与面板中的 outliner 完全一致——可编辑、可无限展开、可拖拽。嵌入的是真实节点引用，不是快照。

用户在 Chat 内嵌 outliner 中编辑了节点后，system reminder 会注入编辑状态（"本会话中提及的节点 X 在 3 分钟前被编辑"），AI 下一轮自然知道该重新读取。

详细设计见 `docs/plans/chat-node-display.md`。

### 浏览器上下文

soma 活在浏览器侧边栏，天然知道用户正在看什么。这个上下文自动注入对话，不需要用户解释。

- 用户在看一篇文章，直接问"这篇文章的要点" → AI 已经知道是哪篇
- 用户在看一个产品页，说"记一下这个定价方案" → AI 知道具体内容
- 用户在看自己之前的笔记，问"帮我扩展一下这个想法" → AI 知道上下文

### 预设召唤

用户主动设置持续运行的指令：
- "每天早上总结昨天的笔记"
- "浏览 AI 相关文章时提醒我相关笔记"
- "每周回顾 #research 下的新增内容"

不是 AI 自作主张推送，而是用户授权的自动化。

## 七、异步任务与多 Agent 协作

### 任务即节点

不需要独立的任务管理系统。节点树本身就是任务系统：

```
"调研 AI Agent 竞品"  #task  status: running
  ├── 搜索主流平台          #task-step  status: done
  ├── 阅读 CrewAI           #task-step  status: running  ← subagent 1
  ├── 阅读 AutoGen          #task-step  status: running  ← subagent 2
  ├── 阅读 LangGraph        #task-step  status: pending
  └── 整理对比表            #task-step  status: pending
```

- `#task` 是 supertag，定义 `status` 字段（pending / running / done / failed）
- 任何节点打上 `#task` 就自动拥有状态
- 状态变更 = `node_edit` 修改字段值，subagent 用现有工具即可操作
- 任务完成的结果 = 子节点，天然在知识图谱中可搜索、可关联

### 异步任务的运行模式

当前架构约束：AI 请求从 Side Panel 进程发出，Side Panel 关闭则进程终止。因此异步任务**不是**服务端后台运行，而是：

- 侧边栏保持开着，AI 自主跑多轮 tool call
- 用户可以去其他标签页做自己的事
- AI 的浏览器操作（如果需要打开页面、抓取内容）放入 **tab group** 隔离，不打扰用户的标签页

Chat UI 需要任务感知视图：
- 当前有哪些 agent 在执行什么任务
- 每个任务的实时进度（checklist 自动更新，因为底层是节点字段变化）
- 已完成任务的结果（内嵌节点展示）

### 多 Agent 协作

异步任务的扩展——多个 Agent 并行执行子任务：
- 主 Agent 拆任务，分给多个 subagent 并行
- Agent 之间可以相互询问信息和通知状态
- 编排层（Orchestrator）协调任务分配和依赖关系
- 异步能力是多 Agent 的前置条件

## 八、设计红线

1. **用户永远可以覆盖 AI 的任何决定**
2. **AI 创建的内容标记来源**
3. **不是每次交互都要求用户判断，但高价值时刻要创造判断机会**
4. **笔记不够多时坦诚说"再积累一些"**
5. **预设召唤需要用户主动设置**
6. **用户可感知的长期记忆必须是节点，不放进黑箱 memory 数据库**
7. **AI 沉淀遵守跟用户一样的节点系统规则，不发明特殊机制**

## 九、演进历程

```
v1  记录工具    "记下来，用得上"           捕获 → 消化 → 重遇
v2  思考工具    "Think where you read"    Think → Connect → Compound
v3  +AI 策略    "Think where you read"    + 认知工作流
v4  AI 照亮     "Think where you read"    Think → Connect → Emerge
v5  人机协作    "Think with your AI"      对话即思考
```

从"记录"到"思考"到"协作"，每次迭代都在把更多的认知工作从用户独自承担，转移到人机共同完成。核心不变的是：**工具让人变强，不是让人觉得工具强。**

各版本详情：

- **v1**：捕获 → 消化 → 重遇（三步线性流）。笔记是对信息的预处理，AI 角色未明确
- **v2**：Think → Connect → Compound。从"记录"升级为"思考"，费曼学习法，AI 辅助
- **v3**：加入"选择是思考的第一步"（高亮即思考）、三条认知工作流（阅读环 / 碰撞环 / 回顾环）
- **v4**：Think → Connect → Emerge。AI 独立成章（"照亮你的思考"）、干/湿分工、设计红线。文档：`product-philosophy.md`
- **v5**：对话即思考。一个意图入口，AI 从工具变为协作伙伴，Outliner 从主界面变为可选深入层。文档：本文件

## 十、未决问题

### 跨会话记忆

1. **跨会话检索的 scope 怎么切** — 用户主动检索偏向全局个人历史。剩下的张力：AI 自动引用时是否也默认全局，还是优先当前上下文再按需升级
2. **AI 什么时候该把旧东西拉回来** — 过于积极像翻旧账，过于保守失去跨会话价值。需要定义触发策略和分档

### 用户控制

3. **控制与可解释性** — 来源默认收起已确定。还需定义最小控制面：查看来源、排除某段历史、撤销 AI 写入的节点、区分过程历史和长期记忆

### 产品定位

4. **Chat 作为主入口，会不会压扁结构思考** — 若用户长期只停留在 Chat，soma 会滑向聊天产品。需要"从对话回到结构"的引导机制
5. **冷启动** — Day 1 用户没有积累。需要回答：Day 1 的即时价值是什么，连续使用 1 周后的复利价值是什么
6. **对外叙事** — 内部可以理解多个维度，对外只能卖一件事。需要收敛一级定位
