# Feature: 节点选中

> 全量规格 | 基于 Tana 双模式选择范式

## 概述

大纲编辑器中存在两种互斥的交互模式：

| 模式 | 用户所见 | 含义 |
|------|---------|------|
| **编辑模式** | 某个节点内有闪烁的文本光标 | 用户正在编辑该节点的文字内容 |
| **全局选中模式** | 一个或多个节点整行高亮，无文本光标 | 用户正在对节点整体进行操作（Esc/框选/多选） |
| **引用单击选中模式** | reference 行出现 `fit-content` 边框，无文本光标 | 用户选中了 reference 本体，准备删除/转换/双击编辑 |
| **无选择** | 无任何高亮或光标 | 空闲状态 |

多选允许用户同时操作多个节点（删除、缩进、移动等），是高效编辑的基础。

## 一、模式切换

```
                    ┌──────────────────┐
                    │    无选择状态      │
                    └────────┬─────────┘
                             │ 点击节点
                             ▼
                    ┌──────────────────┐
          ┌────────│    编辑模式        │────────┐
          │        │ (光标在节点内)      │        │
          │        └──────────────────┘        │
          │              │                      │
          │   Esc        │ Shift+↑/↓            │ Shift+↑/↓
          │              │ (首次按键)            │ (后续按键)
          │              ▼                      │
          │        ┌──────────────────┐        │
          │        │    选中模式        │◄───────┘
          │        │ (整行高亮,无光标)   │
          │        └──────────────────┘
          │              │        │
          │   Enter/     │        │ Esc
          │   输入字符    │        │
          │              │        ▼
          │              │  ┌──────────────────┐
          └──────────────┘  │    无选择状态      │
                            └──────────────────┘
```

### 编辑模式 → 选中模式

| 触发 | 行为 |
|------|------|
| **Esc**（无下拉菜单打开） | 光标消失，当前节点保持高亮，编辑器变为只读 |
| **Shift+↑/↓**（首次） | 清除浏览器文本选区，退出编辑，当前节点成为选择锚点 |

### 选中模式 → 编辑模式

| 触发 | 行为 |
|------|------|
| **↑**（无修饰键） | 清除选区，编辑选区上方相邻可见节点，光标在**文本末尾** |
| **↓**（无修饰键） | 清除选区，编辑选区下方相邻可见节点，光标在**文本开头** |
| **Enter** | 聚焦第一个选中节点，光标在文本末尾 |
| **输入可打印字符** | 聚焦第一个选中节点，字符追加到文本末尾 |

> 光标位置约定：↑ 到末尾、↓ 到开头，符合用户从上/下方进入节点的直觉。

### 选中模式 → 无选择

| 触发 | 行为 |
|------|------|
| **Esc** | 清除所有选择 |

### 无选择 → 编辑模式

| 触发 | 行为 |
|------|------|
| **点击任意节点** | 聚焦该节点，光标由浏览器根据点击坐标放置 |

> FieldRow 补充：普通单击遵循“编辑优先”——点击 `field name` 进入字段名编辑，点击 `value` 区进入对应值编辑/交互态；不再把普通单击解释为 tuple 选中。多选仍由 `Cmd/Ctrl+Click`、`Shift+Click`、拖选与 `Shift+↑/↓` 扩展完成。
> 手势优先级补充：`Cmd/Ctrl` 与 `Shift` 选择手势在 FieldRow/ContentRow 共享同一 pointer-intent 解析，保证 node 类型不影响多选行为。

### Esc 三级退出链

连续按 Esc 依次经过：**编辑模式 → 选中模式 → 无选择**。如果当前处于 Zoom（聚焦某节点的子树视图），无选择状态下按 Esc 还会退出 Zoom（PanelStack.pop）。

### Escape 键优先级

1. 如果有下拉菜单打开（#tag / @reference） → 关闭菜单（现有行为）
2. 如果在编辑模式（无下拉菜单）→ 退出编辑，当前节点变为选中
3. 如果在选中模式 → 取消所有选中

## 二、鼠标交互

### 2.1 普通点击

1. **mousedown 阶段**：记录文本偏移量（光标位置），但**不进入编辑模式**。这允许后续拖动触发 drag-select
2. **click 阶段**：如果刚结束拖选操作（`justDragged` 标志），忽略这次点击
3. 如果存在多选（≥2 节点），先清除多选
4. 将点击节点设为唯一选中项
5. **点击文本区域**：进入编辑模式，光标位置由 mousedown 记录的偏移量恢复
6. **点击非文本区域**（左侧 padding、bullet 区域）：进入编辑模式，光标放在文本末尾

> 延迟焦点设计：mousedown 不挂载 TipTap 编辑器，避免编辑器捕获后续 mousemove 事件导致 drag-select 失效。

### 2.2 Cmd+Click（多选切换）

- 节点已选中 → 取消选中
- 节点未选中 → 加入选中集合
- 选中结果 ≥ 2 个节点时，自动退出编辑模式进入选中模式
- **子树合并吸收规则**：
  - 点击已选父节点的后代 → **忽略**（已被祖先覆盖）
  - 点击已选后代节点的祖先 → 移除所有已选的后代，只保留祖先（合并吸收）

### 2.3 Shift+Click（范围选择）

- 从**锚点节点**到点击节点之间的所有**可见节点**全部选中
- 可见节点 = 未被折叠隐藏的节点，按深度优先遍历排列
- 仅保留根级选中：如果父子节点都在范围内，只保留父节点
- 退出编辑模式，进入选中模式

### 2.4 双击

- 普通节点：浏览器默认行为（选中单词）
- 引用节点：进入编辑模式，光标在文本末尾（单击引用节点不进入编辑，双击才会）

## 三、拖动选择（Drag Select）

### 启动条件

- 左键按下（`button === 0`），无 Cmd/Shift/Ctrl/Alt 修饰键
- 点击位置必须在容器内、在某个节点行上
- 不在 button/input/`[role="button"]` 等交互元素上
- 鼠标垂直移动超过 **5px** 阈值后激活

### 文本区域智能判断

从文本区域（`.editor-inline`、`.node-content`、`[contenteditable]`）开始拖动时，有三种分支：

1. **仍在同一节点的文本区域上** → 让浏览器原生文本选择工作（不激活拖选）
2. **仍在同一节点但移到非文本区域**（padding 等） → 激活拖选
3. **移到不同节点** → 激活拖选

从非文本区域开始拖动时，超过 5px 阈值即激活拖选。

> 设计意图：在文字上拖动大概率是选择文字，移到其他节点或 padding 才说明意图是多选。
> NodePanel 补充：drag-select 根列表包含同级 `content row + field row`，所以从内容节点拖过字段区域时，字段 tuple 会进入选区（不再被过滤）。

### 拖选过程

1. 清除浏览器文本选区（`window.getSelection()?.removeAllRanges()`）
2. 容器添加 `select-none` CSS 类防止文本选择伪影
3. 起始节点被选中并设为锚点（`setSelectedNodes` 清除 focusedNodeId）
4. 鼠标移动过程中，实时更新选区：选中从锚点到当前悬停节点之间的所有可见节点
5. 安全检查：`mousemove` 中检测 `me.buttons !== 1`（左键已释放但 mouseup 未触发），自动 cleanup

### 拖选结束

- 释放鼠标后，选区保持
- 设置共享标志 `dragState.justDragged = true`
- 通过 `setTimeout(0)` 在下一个微任务中重置为 `false`
- OutlinerItem 的 `handleContentClick` 检查此标志，跳过点击处理

### 实现架构

```
mousedown (document, capture)
  → 记录 startY、startNodeId、startedOnText
  → 注册 mousemove + mouseup 到 document（动态监听器）

mousemove (document, 动态)
  → 阈值检查 (5px)
  → 文本区域智能判断
  → 激活后：select-none + setSelectedNodes + 实时范围更新

mouseup (document, 动态)
  → justDragged 标志 + setTimeout(0) 重置
  → cleanup（移除动态监听器、恢复样式）
```

- 使用**动态** document 级监听器（mousedown 时注册，mouseup 时移除），而非永久监听
- `justDragged` 是模块级共享状态，OutlinerItem 通过 `import { dragState }` 访问
- 不依赖 React 事件系统，避免 contenteditable 事件捕获问题

## 四、键盘快捷键

### 4.1 编辑模式下

| 按键 | 行为 |
|------|------|
| **Esc**（无下拉菜单） | 退出编辑 → 进入选中模式（当前节点保持高亮） |

| **↓** | 光标移到下一可见节点，光标在**文本开头** |
| **Shift+↑/↓**（首次） | 退出编辑进入选中模式，当前节点为锚点 |

### 4.2 选中模式下

| 按键 | 行为 |
|------|------|
| **Esc** | 清除所有选择 → 无选择状态 |
| **↑** | 退出选中 → 编辑选区上方相邻节点（光标在末尾） |
| **↓** | 退出选中 → 编辑选区下方相邻节点（光标在开头） |
| **Shift+↑** | 从锚点向上扩展选区一个节点 |
| **Shift+↓** | 从锚点向下扩展选区一个节点 |
| **Enter** | 编辑第一个选中节点，光标在末尾 |
| **可打印字符** | 编辑第一个选中节点，字符追加到末尾 |
| **Cmd+A** | 选中当前根节点的所有直接子节点 |
| **Backspace / Delete** | 批量删除所有选中节点，焦点移到最上方被删节点的上一个节点 |
| **Tab** | 批量缩进所有选中节点 |
| **Shift+Tab** | 批量取消缩进 |
| **Cmd+Shift+D** | 批量复制所有选中节点，完成后清除选区 |
| **Cmd+Enter** | 批量切换 Checkbox（有任何未完成 → 全部完成；全部已完成 → 全部取消） |

## 五、子树选择

### 核心原则

**选中父节点 = 隐式选中所有后代。** 选择集合只存储根级选中节点，不存储被祖先覆盖的后代。

### 判断节点是否被选中

沿所有权链（parent chain）向上遍历，只要任一祖先被选中，该节点即为"被选中"状态。

### 对各操作的影响

| 操作 | 处理逻辑 |
|------|---------|
| Cmd+Click 后代 | 后代已被祖先覆盖，忽略操作 |
| Cmd+Click 祖先 | 移除已选的后代，只保留祖先（合并吸收） |
| 范围选择 | 计算完范围后过滤：父子都在范围内时只保留父 |
| Shift+Arrow 扩展 | 同样的合并吸收逻辑 |
| 批量删除 | 只删除根级选中节点（后代随父节点一同删除） |
| 批量缩进/取消缩进 | 只操作根级选中节点 |

## 六、选中态视觉呈现

### 6.1 普通节点 — 子树遮罩 + 独立行高亮（Tana-style）

两层叠加实现 Tana 风格的选中视觉：

1. **子树遮罩**：`--selection` (`rgba(139,92,246,0.08)`)，无边框，仅展开时显示，覆盖整个子树区域，子节点作为连续整体，`rounded-b-sm`（底部圆角）
2. **直接行高亮**：`--selection-row` (`#E8E0FA` 不透明) + `border border-primary/[0.15]`，仅覆盖直接选中的行，不透明色避免与子树遮罩叠加，边框营造荧光感

> 行高亮有 1px 垂直内缩 (`top: 1; bottom: 1`)，相邻直接选中行之间有 2px 可见间隙。
> 子树遮罩有 1px 底部内缩 (`bottom: 1`)，与下一兄弟节点保持一致的 2px 间隙。
> 字段行（FieldRow）与内容行（content node）共享同一行高亮视觉配方；字段行内容层（name/value）必须在遮罩之上，避免选中后值区文本不可见。
> 字段行选中遮罩左边界对齐到 content row 基线（FieldRow 内部做 `left: -4px` 补偿 chevron-bullet gap）。

**视觉效果**：
- 单选展开的父节点：子树区域被浅紫色连续遮罩覆盖，父节点行颜色更深（不透明 `#E8E0FA`）
- 多选多个节点：每个选中节点各自独立行高亮，相邻行之间有 2px 间隙
- 子节点不单独高亮，由子树遮罩统一覆盖，颜色更浅，形成一个连续整体

### 6.2 引用节点

- 使用与普通节点相同的子树边框 + 行高亮方案
- Reference 节点选区使用显示层级（display hierarchy）而非 `_ownerId` 链判断
- 修复：Shift+Arrow 跨 reference 节点扩展选区不再振荡

### 6.3 暗色模式（待实现）

- 子树遮罩：primary 20% 不透明度
- 节点行高亮：primary 40% 不透明度

## 七、引用节点的特殊选择行为

| 交互 | 普通节点 | 引用节点 |
|------|---------|---------|
| **单击** | 进入编辑模式，出现文本光标 | 仅选中（只读，紧凑蓝色边框），不进入编辑 |

## 八、锚点机制

"锚点"是范围选择和 Shift+Arrow 的参考起点：

| 触发 | 锚点设为 |
|------|---------|
| 普通点击节点 | 点击的节点 |
| Shift+↑/↓ 首次按键 | 当前编辑中的节点 |
| 拖选开始 | 拖选起始节点 |

- Shift+Click 的范围 = 从锚点到点击目标之间的所有可见节点
- Shift+Arrow 的扩展 = 从锚点向目标方向逐步扩展

---

## 九、批量操作执行顺序

不同操作需要不同的遍历顺序以保证正确性：

| 操作 | 遍历顺序 | 原因 |
|------|---------|------|
| **删除** | 从下往上 | 避免先删上方节点导致下方节点索引偏移 |
| **缩进** | 从上往下 | 上方节点先缩进成为前一兄弟的子节点，下方节点再缩进时能跟随 |
| **取消缩进** | 从下往上 | 下方节点先取消缩进，避免影响上方节点的父子关系 |
| **复制** | 从下往上 | 副本插入在原节点下方，从下往上保证位置正确 |
| **切换 Checkbox** | 顺序无关 | 各节点独立切换 |

---

## 十、Zoom 切换与选择

当用户切换 Zoom（聚焦到某个节点的子树视图）时：

- **自动清除所有选择**：防止跨视图的残留选择
- 无选择状态下按 **Esc** 会退出 Zoom（PanelStack.pop），回到上一层视图

## 实现考量

### 多选与 FieldValueOutliner 的关系

- 多选仅在主 OutlinerView 中有效
- FieldValueOutliner 内的节点不参与多选
- 在 FieldValueOutliner 上下文中，Shift+Arrow 应触发 `onNavigateOut` 而非扩展选区

### 可见节点范围计算

Shift+Click / 拖动选择需要计算两个节点之间的可见节点列表。使用 `tree-utils.ts` 中的 `flattenVisibleNodes` 工具函数，结合 `expandedNodes` 状态确定哪些子树是展开的。

### 拖动选择的 document 级监听

contenteditable 元素会捕获鼠标事件，导致父容器上的 React onMouseMove 不可靠触发。拖选使用**动态** document 级监听器（mousedown 时注册，mouseup 时移除）。

关键设计：焦点入口从 mousedown 延迟到 click。mousedown 阶段仅记录文本偏移量，不挂载 TipTap 编辑器——这避免编辑器捕获后续 mousemove 事件。拖选完成后通过共享 `dragState.justDragged` 标志抑制 click 处理，防止干扰选区。

## 实现范围

### Phase 1（基础单选交互）

| 功能 | 说明 |
|------|------|
| Escape 退出编辑 → 选中 | Esc 三级退出链前两级 |
| 选中模式 ↑/↓ 导航 | ↑ 光标在末尾，↓ 光标在开头 |
| 选中模式 Enter 回编辑 | 光标在文本末尾 |
| 选中模式可打印字符 | 聚焦首个选中节点，字符追加到末尾 |
| Shift+↑/↓ 从编辑进入选中 | 首次按键退出编辑，当前节点为锚点 |

### Phase 2（多选 + 拖选）

| 功能 | 说明 |
|------|------|
| Cmd+Click 多选 | 含子树合并吸收规则 |
| Shift+Click 范围选中 | 锚点到目标之间可见节点，根级过滤 |
| Shift+Arrow 扩展选区 | 从锚点持续扩展 |
| 拖动选择 | 文本/非文本区域区分启动，5px 阈值，document 级监听 |
| Cmd+A 全选 | 编辑模式=选文字，选中模式=选所有顶层节点 |

### Phase 3（批量操作 + 视觉升级）

| 功能 | 说明 |
|------|------|
| Backspace/Delete 批量删除 | 从下往上遍历 |
| Tab / Shift+Tab 批量缩进 | 缩进从上往下，取消缩进从下往上 |
| Cmd+Shift+D 批量复制 | 从下往上遍历 |
| Cmd+Enter 批量 Checkbox | 顺序无关 |
| 独立行高亮 | Tana-style per-row highlight（ancestorSelected prop 传递） |
