# #50 `#query` 选中 tag 后触发词未清理（如 `#per` 残留）

**状态**: 📋 待认领  
**负责人**: —  
**优先级**: P1  
**关联**: editor / hashtag trigger

---

## 问题描述

在节点文本中输入 `#per`，按 `Enter` 选择下拉最上方的 `#Person` 后，tag 会被正确应用，但原文本中的 `#per` 仍保留在节点内容里。

该问题已经多轮修复尝试，依然稳定复现，说明当前“按 range 删除”的思路存在系统性缺陷（range/caret 与当前 DOM 状态不同步）。

## 复现步骤

1. 聚焦任意普通节点（非 reference）。
2. 输入 `#per`。
3. 按 `Enter` 选中下拉首项 `#Person`（或鼠标点击）。
4. 观察节点文本。

## 预期结果

- tag 应用成功；
- 文本中的触发词（`#per`）应被删除；
- 光标保留在触发词原位置。

## 实际结果

- tag 应用成功；
- `#per` 残留在文本中。

## 已尝试方案（失败）

| # | 方案 | 结果 | 备注 |
|---|------|------|------|
| 1 | 依据 `hashRangeRef {from,to}` 删除 | 失败 | range 常漂移 |
| 2 | 依据当前 caret 前缀正则 `/#...$/` 删除 | 失败 | 某些时序拿不到正确 caret |
| 3 | fallback 到 `lastIndexOf(#query)` 删除 | 失败 | query 变化时仍会错位 |
| 4 | 强制文本替换兜底（plain text） | 仍有残留 | 说明触发时机/编辑实例引用也有问题 |

## 建议接手方向

1. 不再依赖“历史 range”，改为“选择 tag 当下”的一次性事务：
   - 读取当前 editor text + selection；
   - 在同一 tick 内计算并删除触发片段；
   - 再应用 tag；
   - 最后统一保存。
2. 打点验证选择路径是否混入了不同 editor 实例（`editorRef` 指向变化）。
3. 增加最小回归测试：
   - `#p`、`#per`、`#person`；
   - 键盘 Enter 选择与鼠标点击选择；
   - 中英文输入法下均通过。
