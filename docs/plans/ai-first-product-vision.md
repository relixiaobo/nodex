# AI-First Product Vision v2

> 从 "工具 + AI 辅助" 翻转为 "人机协作"

## 一、核心洞察

用户不需要更好的笔记工具。用户需要一个**协作伙伴**——记得住所有东西、想得起该想的、说得出有用的话，并且在对话中帮用户变得更清醒。

节点树是共享的思考痕迹，Chat 是协作的界面。

## 二、产品哲学演进

### 从 "Think where you read" 到 "Think with your AI"

v1 产品哲学（`product-philosophy.md`）围绕 **Think → Connect → Emerge** 三步循环：记录 → 组织 → 涌现。每一步依赖用户主动操作。

这个模型的局限：
- **Connect 的成本太高**——愿意手动组织信息的用户 < 1%
- **Emerge 来得太慢**——需要足够积累才能涌现，大部分用户等不到
- **三步是串行的**——用户必须先记录，再组织，再等待涌现

AI-First 模式下，这三件事**在对话中同时发生**：
- 用户带着模糊的想法来对话 → **Think**
- AI 拉入相关笔记，发现连接 → **Connect**
- 用户看到意想不到的关联 → **Emerge**

不再是三步循环，而是一次对话中的一体化体验。

### 保留的核心理念

以下理念在 AI-First 中不变，反而被强化：

- **"工具应该让人变强，而不是让人觉得工具很强"** → AI 协作的设计目标是用户获得新理解，不是 AI 展示能力
- **"知识组织没有客观正确答案"** → AI 的组织可能"错误"，但错误可能是惊喜
- **"Outliner = 共享画板"** → 画板从 UI 概念升级为数据概念，人和 AI 都在上面创作
- **"正反馈是成功的关键"** → 对话本身就是即时反馈，不需要等三个月

### 演进的理念

| v1 理念 | v2 演进 | 原因 |
|---------|---------|------|
| "AI 只在用户召唤时行动" | AI 可在用户授权范围内主动行动 | 用户设置规则 = 预设召唤，AI 主动创建节点 = 协作 |
| "Think → Connect → Emerge 三步循环" | 对话即思考，三者一体 | 对话中同时发生记录、连接、涌现 |
| "AI 不生成新观点" | AI 可以记录和提炼，但用户保留判断权 | 对话中的洞察如果不被捕捉就丢失了 |
| "标签是索引，不是分类" | AI 辅助建立结构，用户审查 | 降低结构化门槛，但保留用户判断 |

## 三、人机协作模型

### 协作关系，不是主从关系

当前模式：用户是主人，AI 是工具。用户说"做"，AI 才做。

新模式：**用户和 AI 都是数据的主人。** 像两个人在同一块白板前讨论——任何一方都可以写字、画图、擦除，不需要另一方许可。

具体表现：
- **用户**可以随时创建、编辑、组织节点
- **AI** 在对话中觉得重要的内容，也直接创建节点
- **AI** 发现连接时，主动建立关联
- **一切可审查、可修正、可删除**——协作不等于失控

### 区分来源，不区分权限

AI 创建的节点标记来源（如一个小标识），用户可以：
- 确认 → 标记消失，变成"自己的"
- 编辑 → 变成共创内容
- 删除 → AI 学到这类内容不需要保留

节点树从"用户的笔记本"变成**"人机共同的思考痕迹"**。

### 对话即思考

用户不需要先记录再组织——对话本身就是思考过程：

```
用户有一个想法/问题/好奇心
  → 跟 AI 对话
    → AI 拉入相关笔记、发现连接、提出新角度
      → 用户做判断（追哪个方向、哪个有关、哪个不对）
        → 有价值的部分沉淀为节点
          → 下次对话时被 AI 拉入上下文
```

每次对话都在丰富共享画板，画板又让下次对话更深入。**飞轮不是 Think → Connect → Emerge，而是 对话 → 沉淀 → 对话。**

## 四、认知摩擦设计

### 为什么需要摩擦

AI 协作中最省力的路径是"让 AI 做完，我看结果"。但这让用户变成 AI 输出的消费者——没有思考，没有成长。

soma 的独特定位：**在正确的地方主动增加认知摩擦。** 不是阻碍用户，而是在正确的时刻邀请用户做一个判断。判断 = 思考 = 成长。

关键：摩擦的形式是**选择题，不是问答题**。不是让用户从零构建，而是在 AI 给出的选项中做判断。

### 两层摩擦

**产品层摩擦**（结构性的，所有用户都会遇到）：
- AI 创建的节点默认带"待审查"标记，用户确认后标记消失
- Chat 中 AI 的建议不自动执行，需要用户确认
- 定期 review 提醒（用户设置的频率）

**AI 层摩擦**（情境性的，AI 基于上下文主动提问）：
- 感觉到对话方向模糊："你想深入哪个方向？"
- 发现矛盾："这跟你之前记的 X 好像有冲突？"
- 发现模式："你最近三次讨论都提到了 Y，值得提炼成一个主题吗？"
- 发现连接："这个观点跟你上周的 Z 笔记有关，要看看吗？"

AI 层摩擦更强大（个性化、基于上下文），但更难设计——需要 AI 有足够的"味觉"知道什么时候该问、什么时候该闭嘴。

### 摩擦的度

| 太少 | 恰到好处 | 太多 |
|------|---------|------|
| AI 做完一切，用户只看结果 | AI 做结构化部分，用户做判断 | 用户手动组织一切 |
| 用户没有成长 | 用户在判断中获得新理解 | 用户疲于操作 |
| ≈ ChatGPT | ≈ soma | ≈ 传统笔记工具 |

## 五、产品架构

### Chat 作为主界面

```
┌─────────────────────────────────┐
│  Chat (主界面/默认层)             │
│  对话式交互，人机协作             │
├─────────────────────────────────┤
│  Node Panel (可选的可视化层)      │
│  用户想直接操作时打开             │
│  大纲、字段、标签 — 全部保留      │
└─────────────────────────────────┤
│  LoroDoc (共享数据层)             │
│  人和 AI 共同的思考痕迹           │
└─────────────────────────────────┘
```

### Chat 内嵌节点渲染

AI 的回答不只是文本，而是可以嵌入活的节点。**渐进式交互**：

1. **默认态：紧凑预览** — 标题 + bullet 列表，和文本自然混排
2. **点击：展开详情** — 在 Chat 内展开 children（mini OutlinerView）
3. **双击/按钮：跳到面板** — 打开到独立 NodePanel 深入操作

三层是同一个节点的三种呈现密度，用户通过自然交互逐步深入。

### 数据原则

- Chat 中嵌入的是**真实节点引用**，不是快照副本
- AI 创建的和用户创建的是同一种节点，区别只在来源标记
- Chat 里看到的和 Panel 里看到的永远是同一份数据

### 预设召唤（替代"不问就给"）

用户主动设置持续运行的指令，AI 按规则执行：
- "每天早上总结昨天的笔记"
- "浏览 AI 相关文章时提醒我相关笔记"（用户主动开启）
- "每周回顾 #research 下的新增内容"

这不是 AI 自作主张推送，而是用户授权的自动化。

## 六、待深入的问题

### 认知摩擦的精细设计

- AI 层摩擦的具体触发条件和表现形式
- 如何让 AI 学会"什么时候该问，什么时候该闭嘴"
- 摩擦频率如何随用户使用阶段调整（新手多一些，老手少一些）

### 节点编辑与模型上下文同步

用户在 Chat 中编辑了 AI 返回的节点，模型上下文如何保持一致。

推荐方向：在 `transformContext` 阶段自动注入当前对话引用过的节点的实时状态，而非追踪操作过程。

### AI 主动创建节点的边界

- 什么样的对话内容值得 AI 主动创建节点？
- 创建频率如何控制（不至于产生大量低质量节点）？
- 用户如何批量审查 AI 创建的节点？

### Taste 学习与摩擦的关系

AI 从用户的确认/修正/删除模式中学习，逐渐减少不必要的摩擦、增加有价值的摩擦。

## 七、与现有架构的关系

当前架构已经很接近：
- Node 层是独立数据层（LoroDoc + CRDT）
- Agent 已有 6 个 node tools（CRUD + search + undo）
- Chat 已是面板级，和 NodePanel 同权
- #agent / #skill 配置全部是节点
- `transformContext` 管线已支持动态上下文注入

翻转不需要重写，而是**重新排列优先级**：
- Chat 从配角变主角
- Node Panel 从主界面变工具箱
- AI 从被动工具变协作伙伴
- 节点树从"用户的笔记"变"人机共同的思考痕迹"

## 八、设计红线（更新）

1. **用户永远可以覆盖 AI 的任何决定** — 协作不等于失控
2. **AI 创建的内容必须标记来源** — 用户要能区分自己写的和 AI 记的
3. **每次交互包含至少一个用户判断点** — 防止用户退化为被动消费者
4. **笔记不够多时坦诚说"再积累一些"** — 不造假
5. **预设召唤需要用户主动设置** — AI 不在未授权的情况下主动推送
